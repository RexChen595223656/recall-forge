# v3.0 锻造台重构 交互原型说明

## 1. 页面结构

### 整体布局

```
┌──────────────────────────────────────────────────────────┐
│ 顶部栏 (h-10, fixed)                                     │
│ [R] RecallForge    今日 3/5 题 🔥        [统计 ▼]       │
├────────────────────┬─────────────────────────────────────┤
│ 知识库 (w-72)       │ 锻造区 (flex-1)                     │
│ fixed, overflow-y   │ overflow-y                          │
│                     │                                     │
│ 搜索框 + 筛选标签   │  选定材料后显示空闲态               │
│ + 添加材料          │  点击开始锻造 → 设置态(折叠)        │
│                     │  确认 → 锻造中(答题→反馈→结果)      │
│ 材料卡片列表        │  或直接进入复习模式                  │
│ (点击选中，高亮)    │                                     │
│                     │  底部常驻全局统计                   │
└────────────────────┴─────────────────────────────────────┘
```

**响应式规则：**
- ≥1024px：两栏布局
- <1024px：左侧面板折叠为左侧抽屉，点击展开
- <768px：知识库底部全宽，锻造区全宽

## 2. 主交互流程

### 2.1 完整锻造流程

```
页面加载
    │
    ▼
[左栏: 加载材料列表]     [右栏: 显示全局概览 + 最近活动]
    │
    ├── 点击材料卡片
    │       │
    │       ▼
    │   [右栏切换到空闲态: 显示该材料统计 + 历史]
    │       │
    │       ├── 点击 [▶ 开始锻造]
    │       │       │
    │       │       ▼
    │       │   [设置态: 快捷按钮 + 高级设置(折叠)]
    │       │       │
    │       │       ├── 点 [使用默认配置开始]
    │       │       │       │
    │       │       │       ▼
    │       │       │   [生成动画] → [答题] ⇄ [反馈] → [结果]
    │       │       │
    │       │       └── 点 [高级设置 ▾] → 展开配置 → 修改 → 开始
    │       │
    │       └── 点击 [♺ 仅复习薄弱点 (N)]
    │               │
    │               ▼
    │           [复习模式: SM-2 卡片流] → [完成]
    │
    └── 搜索/筛选 → 材料列表过滤 → 点击 → 同上
```

## 3. AI 交互节点详解

### 节点 1：锻造区空闲态

| 项目 | 说明 |
|------|------|
| **触发** | 用户点击左侧材料卡片 |
| **输出区域** | 锻造区全宽 |
| **核心组件** | 掌握度进度条 + 材料统计卡片 + 快捷按钮 + 历史记录列表 |

**状态流：**
```
未选材料(显示全局概览) → 点击材料 → 加载材料统计 → 展示空闲态
                                              → 加载失败 → 提示"材料不存在或已被删除"
```

**兜底设计：**
- 材料被删除后锻造区自动回到全局概览
- 材料无任何数据时，掌握度为 0%，显示"尚未开始学习"

### 节点 2：AI 批量生成（生成动画）

| 项目 | 说明 |
|------|------|
| **触发** | 空闲态点击"开始锻造"并确认设置 |
| **输出区域** | 锻造区中央 |
| **响应样式** | 三圆点跳动 + 文字轮播 |

**状态流：**
```
触发 → [生成中: "分析材料内容"→"提取关键知识点"→"生成题目选项"→"校验答案准确性"]
     → 完成 → 答题态(首题)
     → 超时(45s) → [错误态: "AI 响应超时" + 重试 + 返回]
     → 取消 → 回到空闲态
```

**兜底设计：**
- 该材料全部 chunk 已用过 → 提示"该材料已无新题可出，建议上传更多内容或选择其他材料"
- AI 返回空数组 → "未能生成题目，请重试"
- JSON 解析失败 → "AI 返回格式异常，请重试"

### 节点 3：答题交互（单选/多选统一）

| 项目 | 说明 |
|------|------|
| **触发** | 生成完成后自动进入 |
| **输出区域** | 进度条 + 题目 + 选项 + 标签 |

**单选：**
```
选项显示态 → hover 边框变绿 → 点击 → 选项 disabled → 判定 → 反馈态
```

**多选：**
```
选项显示态(带checkbox) → 点击切换勾选 → 点击[确认选择] → 判定 → 反馈态
                     → 未选任何项时确认按钮 disabled
```

**关键规则：**
- 选项文本格式：`A. 内容` — 以字母+.+空格开头
- 提取答案：`option.charAt(0).toUpperCase()` → `"A"`
- 判定：`正确答案.startsWith(用户选择)` — 方向不能反

### 节点 4：即时反馈

| 项目 | 说明 |
|------|------|
| **触发** | 提交答案后 |
| **输出区域** | 判断标题 + 题目回顾 + 双列对比 + 解析 + 操作按钮 |

**双列对比布局：**
```
[你的选择]            [正确答案]
B. AI可行性评估        C. 模型训练调优
(红色背景=错误)        (绿色背景=正确)
```

**操作按钮：**
- 非最后一题：`[下一题 →]`
- 最后一题：`[查看结果]`

### 节点 5：锻造结果

| 项目 | 说明 |
|------|------|
| **触发** | 最后一题完成后 |
| **输出区域** | 总分 + 逐题回顾列表 + 操作按钮 |

**逐题回顾格式：**
```
✓ 第1题 — 正确
  你的答案: A — 正确答案: A

✗ 第2题 — 错误
  你的答案: B — 正确答案: C
  解析: ...
  状态: 已加入复习队列
```

**操作按钮：**
- `[再来一轮]` → 回到空闲态，可重新开始
- `[返回材料页]` → 回到空闲态

**自动行为：**
- 错题异步调 `enrollQuestion`
- 结果页出现时刷新左栏材料掌握度数据

### 节点 6：复习模式

| 项目 | 说明 |
|------|------|
| **触发** | 空闲态点击"仅复习薄弱点" |
| **输出区域** | SM-2 卡片流 |

**状态流：**
```
进入复习 → 获取到期卡片(GET /api/review/due?material_id={id})
         → [卡片1: 题目+选项 → 查看答案 → 评分 → 下一张]
         → [卡片N] → 完成页
```

**与 v2 复习页的差异：**
- 不再跳转到 `/review`
- 只加载当前选中材料的到期卡片（不是全局的）
- 评分按钮 + SM-2 参数展示不变

**空态：**
- 该材料无到期卡片 → "暂无待复习的题目，都掌握了！"

### 节点 7：知识库空态

| 项目 | 说明 |
|------|------|
| **触发** | 首次使用，无材料 |
| **输出区域** | 左栏空态 + 右栏引导 |

**左栏（知识库）：**
```
┌─────────────┐
│ 知识库为空   │
│              │
│ [上传材料]   │
│ [试试示例]   │
└─────────────┘
```

**右栏（引导）：**
```
┌──────────────────────────────┐
│       欢迎使用 RecallForge    │
│                              │
│  三步开始：                   │
│  1. 上传学习材料              │
│  2. AI 自动出题               │
│  3. 间隔重复巩固记忆          │
│                              │
│  [上传第一份材料]             │
│  [一键导入示例体验]           │
└──────────────────────────────┘
```

## 4. AI 交互细节总表

| 交互点 | 触发方式 | 响应样式 | 加载处理 | 兜底展示 | 反馈机制 |
|--------|---------|---------|---------|---------|---------|
| 材料统计 | 点击材料 | 掌握度条+数字 | 骨架卡片 | 不存在→回全局 | — |
| AI生成 | 点击开始 | 圆点动画+文字轮播 | 文字轮播 | 超时→错误+重试 | 取消按钮 |
| 答题单选 | 点击选项 | 边框变色→禁用 | — | 本地判定兜底 | 即时变色+提交 |
| 答题多选 | 勾选+确认 | checkbox切换 | — | 本地判定兜底 | 确认按钮+提交 |
| 反馈展示 | 判定后 | 双列对比卡片 | — | — | 下一题/查看结果 |
| 锻造结果 | 最后一题后 | 总分+逐题列表 | — | — | 自动加入复习 |
| 复习评分 | 查看答案后 | 四级按钮+后果 | — | 跳过按钮 | SM-2更新 |

## 5. 关键状态标注

### 5.1 掌握度进度条

| 掌握度范围 | 颜色 | 含义 | CSS |
|-----------|------|------|-----|
| 0% | #374151 (gray-700) | 未开始 | bg-gray-700 |
| 1-39% | #ef4444 (red-500) | 薄弱 | bg-red-500 |
| 40-79% | #eab308 (yellow-500) | 进行中 | bg-yellow-500 |
| 80-100% | #22c55e (green-500) | 已掌握 | bg-green-500 |

### 5.2 锻造按钮状态

| 状态 | 文案 | 行为 |
|------|------|------|
| 默认 | ▶ 开始锻造 | 使用默认配置生成 5 题 |
| 薄弱点有数据 | ♺ 仅复习薄弱点 (N) | N 为到期卡片数 |
| 薄弱点无数据 | ♺ 仅复习薄弱点 — 暂无 | disabled |

### 5.3 多选 vs 单选检测

- 前端通过 `q.answers && q.answers.length > 1` 判断多选
- 不依赖后端返回的 `question_type` 字段
- 混合出题时，每道题独立判定显示单选 UI 还是多选 UI
