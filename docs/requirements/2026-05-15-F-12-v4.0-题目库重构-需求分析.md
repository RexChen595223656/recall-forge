# v4.0 PRD — 题目库重构 + 双闭环体系

> 版本：v4.0 | 日期：2026-05-15 | 类型：架构重构

## 一、背景与问题

v3.x 的核心流程是"锻造→出题→答题"，每次答题前需要等 AI 生成 60-90 秒。三个核心问题：

1. **等待不可接受**：一天刷 3 轮 = 3-4 分钟干等
2. **正确率只降不升**：累计计算，早期糟糕表现永久拖累
3. **错题和复习概念混淆**：错题集 = SM-2 复习队列，用户分不清"重新做"和"复习巩固"

## 二、目标

- 上传材料后自动出题，刷题零等待
- 正确率双维度（累计 + 近期），用户能看到进步
- 错题重做和间隔复习概念分离，各有独立闭环
- 标签维度有实际选题作用

### 竞品参考

| | Anki | Quizlet | 本产品 |
|---|------|---------|--------|
| 题目来源 | 用户手动制卡 | 用户手动制卡 + 社区共享 | AI 自动生成 |
| 复习算法 | SM-2 | 简单间隔 | SM-2（退为排序建议） |
| 错题处理 | 标记为"重来" | 重新加入学习 | 独立错题重做闭环 |
| 毕业机制 | 无（永不到期也可复习） | 无 | interval > 21天 + reps >= 3

## 三、架构变更

```
v3.x:                       v4.0:
上传 → 每次手动锻造(等)       上传 → 自动异步出题 → 题目库就绪
  ↓                              ↓
等 AI 出题(60-90s)           选维度 → 筛题 → 开始(0 秒)
  ↓                              ↓
答题                         答题完成 → 错题进入"错题重做"列表
  ↓                                        ↓
结果                        连续做对 3 次 → 攻克 ✓
```

## 四、功能详述

### F1：上传自动出题

上传材料后，后台异步调用 AI 生成题目，不阻塞用户操作。

**流程**：
1. 用户上传材料 → 立即返回成功，材料卡片出现
2. 后台开始异步出题，材料卡片显示进度指示器："出题中... 3/8"
3. 出题完成，卡片显示："8 题可用"
4. 出题失败，卡片显示："出题失败，点击重试"

**出题数量**：
- 首次生成：`count = min(total_chunks * 3, 30)`
- 1 chunk → 3 题，3 chunk → 9 题，10+ chunk → 30 题
- 这是题目库总量，用户答题时从库中抽取（数量由用户选）

**技术实现**：
- 后端 `/api/materials/{id}/generate` 接口（同步或异步）
- 前端上传后自动调用，不阻塞 UI
- 出题进度通过 SSE 或状态轮询

### F2：题目库模式

用户不再"先出题再答题"，而是"从题目库选题答题"。

**空闲态改造**：
```
┌─────────────────────────────┐
│  材料标题            80%    │
│  ████████████░░░ 最近 90%  │
│  15 题可用 · 已覆盖 3/3 知识点 │
│                             │
│  [知识点筛选]               │
│  [不限][概念][架构][实践]     │
│                             │
│  数量: [5] [10] [全部(15)]  │
│  模式: [原文] [拓展]         │
│  反馈: [即时] [测验]         │
│                             │
│  [开始答题]  ← 立即开始，无等待 │
│                             │
│  [补充题目]   ← 已无新题时禁用 │
├─────────────────────────────┤
│  错题重做（3题待攻克）       │  ← 独立入口
│  间隔复习（2题到期）         │  ← 独立入口
├─────────────────────────────┤
│  全部题目（15题） 展开查看    │
│  最近答题记录               │
│  知识标签                   │
└─────────────────────────────┘
```

**关键变化**：
- "开始锻造" → "开始答题"（语义从"生成"变为"选择已有"）
- 数量从"告诉 AI 出多少"变为"从题目库里抽多少"
- 新增"补充题目"按钮（已覆盖 < 总知识点时可用）

### F3：题目补充

当题目库的题被刷完或用户想要更多题时，可以补充生成。

**规则**：
- 已覆盖 < 总知识点 → "补充题目"按钮可用
- 已覆盖 = 总知识点 → "已无新题可出"
- 补充时只从新 chunk 出题（避免重复）
- 补充数量同上限公式

### F4：正确率双维度

**改动**：始终同时显示两个正确率，不管是否相等。

```
正确率 45% · 最近 80% ↑
```

- 累计正确率：所有时间
- 近期正确率：最近 30 天（后端新统计维度）
- 近期 > 累计 → 绿色箭头（在进步）
- 近期 < 累计 → 灰色箭头（在退步）

**计算规则**：
- 近期 = 最近 30 天的 Attempt 统计
- 如果 < 10 次答题，则用最近 3 轮

### F5：错题重做（独立闭环）

**概念**："这道题我上次做错了，重做一遍看看有没有真搞懂"

**入口**：材料页 → "错题重做（N题待攻克）"

**流程**：
1. 进入错题重做模式
2. 显示题目（不给答案），用户选择
3. 选对 → "✓ 正确"
4. 选错 → "✗ 错误"（不显示正确答案）
5. 下一题
6. 全部完成后统计

**攻克条件**：
- 连续 3 次做对 → 标记为"已攻克"
- 为什么是 3 次：1 次可能是蒙对，2 次可能是短期记忆，3 次说明确实掌握了
- 已攻克题目移出错题列表
- 用户可查看"已攻克"列表

**状态流转**：
```
做错 → 待攻克（streak=0）
  ↓ 做对
待攻克（streak=1）
  ↓ 做对
待攻克（streak=2）
  ↓ 做对
已攻克 ✓（移出列表）
  ↓ 做错（任何一步）
streak 重置为 0
```

**数据模型**：Question 表新增 `challenge_streak` 字段（记录连续做对次数）和 `mastered` 字段（是否已攻克）。

### F6：间隔复习（SM-2 独立闭环）

**概念**："SM-2 算法说该复习了，看看还记得多少"

**入口**：材料页 → "间隔复习（N题到期）"

**流程**：保持现有 SM-2 流程不变——看答案 → 自评 0-3 → 更新间隔。

**毕业条件**：
- `interval_days > 21` 且 `repetitions >= 3` → 标记为"稳定"
- 两个条件同时满足：间隔足够长（记忆已巩固）且连续成功次数足够（不是偶然）
- 毕业后的卡片不出现在到期列表，但仍可在"稳定卡片"中手动复习
- 手动复习评分 < 2 → 重置 is_stable = false，重新进入复习循环

**数据模型**：ReviewCard 新增 `is_stable` 字段（默认 false）。

### F7：两个入口的差异化展示

| | 错题重做 | 间隔复习 |
|---|---|---|
| 标题 | 错题重做（3题待攻克） | 间隔复习（2题到期） |
| 子标题 | "不给答案，检验是否真会" | "看答案自评，巩固记忆" |
| 是否显示答案 | 否 | 是（点击后） |
| 评分方式 | 选择选项（对/错） | SM-2 0-3 分自评 |
| 闭环信号 | 连续 3 次对 → 已攻克 | interval > 21天 + reps ≥ 3 → 稳定 |
| 颜色 | 橙色系 | 蓝色系 |

### F8：全局复习

跨材料聚合复习，两个独立入口：
- "错题重做（全部·N题）"：所有材料中 mastered=false 且有错误 Attempt 的题目，按 challenge_streak 升序
- "间隔复习（全部·M题）"：所有材料中 is_due=true 且 is_stable=false 的卡片，按 next_review 升序
- 每道题标注来源材料名称
- 交互逻辑与单材料模式完全相同，仅数据来源跨材料

### F9：标签选题

标签云从"仅查看"变成"选题维度"。

**交互**：
- 点击标签 → 选中高亮
- 多选支持（AND 逻辑）
- 选择后"开始答题"从对应标签的题目中抽取
- "全部题目"也按标签过滤

**默认行为**：
- 不选任何标签 = 从全部题目中随机抽
- 选了标签 = 只从该标签题目中抽，不够则提示"该知识点仅 N 题，全部抽取"
- 多选 AND 交集为空 → 提示"没有同时包含这些知识点的题目"，阻止答题
- 选中标签后"全部题目"区实时过滤，可折叠预览

## 五、不在本次范围

- 题目编辑器（手动修改/新增题目）
- 自定义出题 Prompt
- 多材料混合刷题
- 排行榜/社交功能

## 六、验收标准总览

| # | 功能 | 验收标准 |
|---|------|---------|
| F1 | 自动出题 | 上传后 30s 内题目库就绪，材料卡片显示进度 |
| F2 | 题目库模式 | 点击"开始答题"后 0 秒等待，直接出题 |
| F3 | 补充题目 | 已覆盖 < 总 chunk → 可补充；= 总 chunk → 禁用 |
| F4 | 正确率 | 始终显示"总 N% · 近期 N%"，带趋势箭头 |
| F5 | 错题重做 | 连续 3 次做对 → 攻克移出；做错 → streak 重置 |
| F6 | 间隔复习 | interval > 21 天 + reps ≥ 3 → 稳定移出 |
| F7 | 差异化入口 | 错题重做和间隔复习各独立入口，文案不同 |
| F8 | 全局复习 | 两个全局入口区分错题/复习 |
| F9 | 标签选题 | 点击标签筛选，进入答题时只出该标签题目 |

| P2 | F3 补充题目 | 依赖 F1 |
| P2 | F8 全局复习 | 锦上添花 |
| P2 | F9 标签选题 | 依赖 F2 |
