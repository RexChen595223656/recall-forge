## 2. AI 功能详细逻辑

### M2.1 AI 动态出题

- **触发条件：** 用户选择材料，点击"开始刷题"
- **输入数据：** RAG 检索的 Top-3 chunk + 历史错题标签（用于调整难度）
- **AI 处理流程：**
  1. 检索 ChromaDB 获取 Top-K chunk
  2. 构建 Prompt（含材料 chunk + 出题要求 + 格式约束）
  3. 调用 DeepSeek API，SSE 流式返回
  4. 前端逐 token 渲染题目
  5. 解析完整题目 JSON（题干、4选项、答案、解析、标签）
- **输出格式：**
  ```json
  {
    "question": "题干",
    "options": ["A. xxx", "B. xxx", "C. xxx", "D. xxx"],
    "answer": "A",
    "explanation": "解析",
    "tags": ["知识点A", "难度:中等"]
  }
  ```
- **异常处理：** JSON 解析失败 → 重试一次 → 降级为纯文本展示
- **兜底方案：** 连续失败 3 次 → 提示用户"AI 暂时无法生成题目，请稍后重试"

### M2.2 即时判定

- **触发条件：** 用户点击选项
- **输入数据：** 用户选择 + 预设答案
- **处理流程：** 前端本地比对，不调后端
- **输出：** 正确(绿)/错误(红) + 展示解析
- **异常处理：** 无预设答案 → 后端补判定 → 降级为"记录已保存"

## 3. 状态矩阵

| 功能点 | 正常态 | 空态 | 加载态 | 错误态 | 边界态 | 兜底态 |
|--------|--------|------|--------|--------|--------|--------|
| 材料上传 | 上传成功，展示进度 | 拖拽上传引导 | 进度条+百分比 | 格式不支持提示 | 超大文件(>10MB)拒绝 | 不支持格式转纯文本尝试 |
| AI出题 | 流式展示题干+选项 | 开始按钮 | 骨架屏+脉冲 | 重试按钮 | 材料不足1个chunk | "请上传更多材料" |
| 答题判定 | 即时变色+解析 | - | - | 判定失败不阻塞 | 快速连点防抖 | 记录原始答案 |
| 复习 | 展示到期题目 | "暂无待复习题目" | 加载中 | 重试 | 全部到期(第一天) | 随机抽取5题 |

## 4. 权限与配置

- 单用户模式，无需登录/权限
- 知识库大小限制：100MB / 100个文档（前端提示）
- 每日题目生成上限：200题（软限制，可配置）
