# 知锻 RecallForge 交互原型说明

## 1. 页面结构

```
┌─────────────────────────────────────────────────────┐
│ 侧边栏 (220px fixed)     │  主内容区                  │
│                          │                           │
│ [R] RecallForge          │  ┌─────────────────────┐ │
│     知锻 · 知识内化引擎   │  │  页面内容            │ │
│                          │  └─────────────────────┘ │
│ ● 知识库                 │                           │
│   管理学习材料             │                           │
│ ○ 上传材料               │                           │
│   添加新内容              │                           │
│ ○ 记忆复习               │                           │
│   间隔重复巩固             │                           │
│                          │                           │
│ AI 驱动 · 间隔重复        │                           │
└─────────────────────────────────────────────────────┘
```

## 2. 交互流程

### 2.1 刷题主流程

```
知识库 → 点击材料 → [设置面板] → 开始出题 → [生成动画] → [答题] → [反馈] → 下一题
                ↑                                    ↓ 答错
                └──────────── 再做一轮 ←── [完成] ←── 自动加入复习队列
```

### 2.2 复习主流程

```
复习页(默认"今日到期") → [卡片] → 查看答案 → [评分] → 下一卡 → [完成]
                            │                                    │
                            └── 全部卡片标签 ←── 查看全部 ←──────┘
```

## 3. AI 交互节点详解

### 节点 1：出题设置面板

| 项目 | 说明 |
|------|------|
| **触发** | 点击知识库中的任意材料 |
| **页面分区** | 标题区(返回/标题) + 设置区(5个配置项) + 操作区(开始出题按钮) |
| **交互组件** | 配置文件选择器（边框按钮组+复选框） |

**状态流：**
```
初始态(默认值) → 用户调整配置 → 点击开始出题 → 切换到生成动画
                                              → 取消 → 返回知识库
```

**默认值设计（黄金路径）：**
- 提取方式：原文提取
- 难度：中等
- 题型：单选 ✓
- 数量：5 题
- 反馈：即时反馈

### 节点 2：AI 批量生成

| 项目 | 说明 |
|------|------|
| **触发** | 点击"开始出题" |
| **输出区域** | 全宽居中卡片 |
| **响应样式** | 呼吸圆点动画 + 文字轮播 + 取消按钮 |

**状态流：**
```
触发 → [生成中:圆点动画 + "分析材料内容" → "提取关键知识点" → "生成题目选项" → "校验答案准确性"]
     → 完成 → 跳转答题(首题)
     → 超时(45s) → [错误态:"AI 响应超时，请稍后重试" + 返回设置按钮]
     → 用户取消 → 返回设置面板
```

**兜底设计：**
- 生成失败 → 显示错误提示 + 返回设置按钮
- chunk 不足 → "材料内容不足，请上传更多内容"
- JSON 解析失败 → "AI 返回格式异常，请重试"

### 节点 3：答题（单选）

| 项目 | 说明 |
|------|------|
| **触发** | 生成完成后自动进入 |
| **输出区域** | 进度条 + 题目 + 4个选项按钮 + 标签 |

**状态流：**
```
答题态 → 点击选项 → disabled(防重复) → 判定 → [反馈态]
```

**交互细节：**
- 点击选项后立即禁用所有按钮
- 前端先本地判定（对比首字母）
- 同时调后端 submitAnswer 记录
- 答错自动调 enrollQuestion 加入复习

### 节点 4：答题（多选）

| 项目 | 说明 |
|------|------|
| **触发** | 当前题为多选题时 |
| **输出区域** | 进度条(标注"多选") + 题目 + 5个选项(带checkbox) + 确认按钮 |

**状态流：**
```
答题态 → 勾选选项(可取消) → 点击确认 → 判定 → [反馈态]
                              → 未选任何选项：按钮 disabled
```

### 节点 5：即时反馈

| 项目 | 说明 |
|------|------|
| **触发** | 提交答案后 |
| **输出区域** | 正确/错误标题 + 题目回顾卡片 + 你的选择 vs 正确答案(双列) + 解析卡片 + 下一题按钮 |

**状态流：**
```
反馈态 → 查看结果 → 点击下一题 → 下题(0ms) / 最后一题点"完成" → [完成页]
```

**颜色语义：** 绿色 = 正确，红色 = 错误，灰色 = 辅助信息

### 节点 6：测验模式回顾

| 项目 | 说明 |
|------|------|
| **触发** | 全部题目答完后 |
| **输出区域** | 总分(正确N/总数M) + 逐题对照列表 + 完成按钮 |

**逐题对照格式：**
```
✓/✗ + 题号. 题目
你的答案: X · 正确答案: Y
解析: ...
```

### 节点 7：复习评分

| 项目 | 说明 |
|------|------|
| **触发** | 点击"回忆一下，点击查看答案"后 |
| **输出区域** | 到期标签 + 题目 + 选项(正确项绿色高亮) + 解析 + 4个评分按钮 + SM-2参数 |

**评分按钮设计（每个显示后果）：**
```
[忘记了 · 明天重来] [有印象 · 1天后] [记住了 · 3天后] [很简单 · 拉长间隔]
```

**SM-2 数据展示：** 间隔 X天 · 难度 X.X · 第 N 次复习

## 4. AI 交互细节总表

| 交互点 | 触发方式 | 响应样式 | 加载处理 | 兜底展示 | 反馈机制 |
|--------|---------|---------|---------|---------|---------|
| 出题设置 | 点击材料 | 设置面板 | — | — | — |
| AI生成 | 点击开始 | 圆点动画 | 文字轮播 | 超时→错误+重试 | 取消按钮 |
| 答题判定 | 点击选项 | 按钮变色+禁用 | — | 本地判定兜底 | 即时变色 |
| 反馈展示 | 判定后 | 双列对比卡片 | — | — | 下一题按钮 |
| 复习评分 | 查看答案后 | 四级按钮+后果 | — | 跳过按钮 | SM-2更新 |
| 测验回顾 | 全部答完 | 总分+逐题列表 | — | — | 完成按钮 |

## 5. 空态设计

| 页面 | 空态文案 | 行动引导 |
|------|---------|---------|
| 知识库(空) | "知识库还是空的" | 上传材料 / 试试示例 |
| 复习(空) | "暂无到期卡片" | 查看全部卡片 |
| 复习(全部空) | "还没有任何复习卡片" | 说明"答错自动加入" |
