# v4.0 API 合约定义

## 新增端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/materials/{id}/generate` | 异步出题 |
| GET | `/api/materials/{id}/status` | 出题状态轮询 |
| GET | `/api/materials/{id}/questions` | 获取题目（已有，新增参数） |
| GET | `/api/review/wrong-questions` | 获取错题列表（支持 ?material_id=N） |
| GET | `/api/review/stable-cards` | 获取稳定卡片列表（支持 ?material_id=N） |

## 修改端点

| 方法 | 路径 | 改动 |
|------|------|------|
| GET | `/api/stats/material/{id}` | 返回 `recent_accuracy` 始终显示 |
| GET | `/api/review/due` | 新增 `is_stable` 过滤，返回 `total_reviews` |
| POST | `/api/review/record` | 毕业判定 `is_stable = True`；稳定卡片评分 < 2 时重置 |
| POST | `/api/quiz/submit` | 存储 `challenge_streak` |

## 数据模型变更

### Question 表新增

```sql
ALTER TABLE questions ADD COLUMN challenge_streak INTEGER DEFAULT 0;
ALTER TABLE questions ADD COLUMN mastered BOOLEAN DEFAULT 0;
```

### ReviewCard 表新增

```sql
ALTER TABLE review_cards ADD COLUMN total_reviews INTEGER DEFAULT 0;
ALTER TABLE review_cards ADD COLUMN is_stable BOOLEAN DEFAULT 0;
```

## 端点详述

### POST /api/materials/{id}/generate

异步出题。

**Request:**
```
POST /api/materials/1/generate
Content-Type: application/json

{
  "exclude_covered": false
}
```

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| exclude_covered | bool | false | true=仅从新 chunk 出题 |

**Response (immediate):**
```json
{
  "status": "generating",
  "max_questions": 9
}
```
`max_questions`：预期生成的题目总数，前端可用此值显示进度后缀（如"出题中... ?/9"）。

**行为：**
1. 立即返回 generating
2. 后台调用 LLM，保存题目
3. 完成/失败后更新状态

---

### GET /api/materials/{id}/status

轮询出题状态。

**Response (generating):**
```json
{
  "status": "generating",
  "max_questions": 9
}
```

注：Thread 方案不支持中间进度推送，前端通过轮询 `status` 字段判断是否完成。

**Response (ready):**
```json
{
  "status": "ready",
  "question_count": 9
}
```

**Response (error):**
```json
{
  "status": "error",
  "error_message": "AI 服务超时，请重试"
}
```

---

### GET /api/materials/{id}/questions

获取题目（已有端点，新增参数）。

**Query Parameters:**

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| count | int | 5 | 返回数量 |
| tags | string | "" | 逗号分隔标签，AND 逻辑 |
| exclude_mastered | bool | false | true=排除已攻克题目 |

**Request:**
```
GET /api/materials/1/questions?count=5&tags=架构,实践
```

**Response:**
```json
[
  {
    "id": 1,
    "question": "RAG架构的核心组件是什么？",
    "options": ["A. 关系型数据库", "B. 向量数据库", "C. 图数据库", "D. 文档数据库"],
    "answer": "B",
    "explanation": "...",
    "tags": ["架构", "RAG"],
    "challenge_streak": 1,
    "mastered": false
  }
]
```

---

### GET /api/review/wrong-questions

获取错题列表（待攻克的题目）。支持全局和单材料两种用法。

**Request (单材料):**
```
GET /api/review/wrong-questions?material_id=1
```
**Request (全局):**
```
GET /api/review/wrong-questions
```

**Response:**
```json
[
  {
    "id": 3,
    "question": "...",
    "options": ["...", "...", "...", "..."],
    "answer": "C",
    "explanation": "...",
    "tags": ["概念"],
    "challenge_streak": 0,
    "mastered": false
  }
]
```

**排序规则：** challenge_streak ASC（streak 低的先做）。

---

### GET /api/review/stable-cards

获取已毕业（stable）的卡片列表。

**Request (单材料):**
```
GET /api/review/stable-cards?material_id=1
```
**Request (全局):**
```
GET /api/review/stable-cards
```

**Response:**
```json
[
  {
    "review_id": 1,
    "question_id": 5,
    "question": "...",
    "options": ["...", "...", "...", "..."],
    "answer": "B",
    "ease_factor": 2.5,
    "interval_days": 30,
    "repetitions": 4,
    "total_reviews": 6,
    "last_review": "2026-05-10T10:00:00",
    "is_stable": true
  }
]
```

**排序规则：** last_review DESC（最近复习的排前）。

### GET /api/stats/material/{id}

**原有响应，新增/修改字段：**

```json
{
  "accuracy": 45.2,
  "recent_accuracy": 80.0,
  "total_questions": 12,
  "wrong_questions": 3,
  "mastered_questions": 2,
  "due_reviews": 2,
  "stable_cards": 5,
  ...
}
```

| 字段 | 说明 |
|------|------|
| recent_accuracy | **始终返回**（不再仅在不等于 accuracy 时） |
| wrong_questions | 待攻克错题数 |
| mastered_questions | 已攻克题数 |
| stable_cards | 稳定卡片数 |

---

### GET /api/review/due

**Query Parameters：**

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| material_id | int | — | 不传=跨材料全局 |

**Response:**
```json
[
  {
    "review_id": 1,
    "question_id": 5,
    "question": "...",
    "options": ["...", "...", "...", "..."],
    "answer": "B",
    "explanation": "...",
    "tags": ["架构"],
    "ease_factor": 2.5,
    "interval_days": 3,
    "repetitions": 1,
    "total_reviews": 3,
    "next_review": "2026-05-18T10:00:00",
    "is_due": true,
    "is_stable": false,
    "material_title": "材料A"
  }
]
```

**过滤规则：**
- is_due = true（next_review <= now）
- is_stable = false
- total_reviews >= 1

### POST /api/review/record

**Request:** （不变）
```json
{
  "question_id": 5,
  "score": 2
}
```

**Response（新增字段）：**
```json
{
  "ok": true,
  "next_review": "2026-05-18T10:00:00",
  "interval_days": 3,
  "is_stable": false
}
```

**稳定卡片手动复习行为：**
- 当 card.is_stable = true 时触发手动复习流程
- score >= 2：保持 is_stable = true，仅更新 last_review（记忆仍牢固）
- score < 2：重置 is_stable = false, interval_days = 1, repetitions = 0（需要重新巩固）
- 不执行 SM-2 计算，直接返回

### POST /api/quiz/submit

**Request:** （不变）
```json
{
  "question_id": 5,
  "user_answer": "B"
}
```

**行为新增：**
- 更新 Question.challenge_streak（答对 +1，答错 = 0）
- 如果 challenge_streak >= 3 → mastered = true

**Response（新增字段）：**
```json
{
  "is_correct": true,
  "correct_answer": "B",
  "explanation": "...",
  "challenge_streak": 2,
  "mastered": false
}
```

---

### GET /api/stats（全局统计）

**新增字段：**
```json
{
  "wrong_questions": 5,
  "due_reviews": 3
}
```

注：`wrong_questions` 跨材料汇总所有 mastered=false 且有错误 Attempt 的题目数，供全局复习入口使用。`due_reviews` 已有，语义不变。
