## 二、M1：上传简化

### 当前问题

1. 三个页签（文本/链接/文件）让用户先选择模式才能操作，增加认知成本
2. 标题需要手动填写，经常留空或用无意义标题
3. 两步流程（输入 → 确认）多一次点击
4. 文件 + 补充说明无法组合（想上传 PDF 同时指定出题方向）

### 设计方案

#### 2.1 布局

```
┌─────────────────────────────────────────┐
│  添加材料                          [×]  │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  粘贴学习材料 / 输入网页链接...   │    │
│  │                                 │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
│  识别为：链接 · 字数提示                │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │        📁 拖拽文件到此处          │    │
│  │    支持 PDF / Markdown / TXT     │    │
│  └─────────────────────────────────┘    │
│  文件：xxx.pdf · 250KB ✓               │
│                                         │
│  标题：[自动识别，可手动修改]            │
│                                         │
│  [确认上传]                             │
└─────────────────────────────────────────┘
```

#### 2.2 智能分流规则

| 文本输入 | 文件 | 行为 |
|---------|------|------|
| 长文本（> 30 字）| 无 | 文本作为学习材料直接出题 |
| URL | 无 | 抓取网页内容出题 |
| 空或短文本 | 无 | 纯补充说明，暂不独立处理 |
| 空 | 有 | 文件内容出题 |
| 有文本 | 有 | 文件为主材料，文本作为出题偏好传给 AI（"重点关注xx"） |

不去猜测用户意图，只用文本长度和格式特征（URL/非URL）做简单判断。

#### 2.3 标题自动识别

| 来源 | 标题规则 |
|------|---------|
| URL | 抓取后取网页 `<title>` |
| 文件 | 文件名去掉后缀 |
| 长文本（> 30 字）| 截取前 30 字 + "..." |
| 空 | 用户手动填写 |

标题字段始终可手动修改。

#### 2.4 去掉两步流程

- 删除 `step` 状态机（"input → confirm"）
- 确认页的预览内容合并到输入页：文本/文件名/URL 实时可见
- 一键上传

#### 2.5 改动文件

| 文件 | 改动 |
|------|------|
| `UploadModal.tsx` | 重写布局，去掉 tabs/step，加标题自动生成 |
| `api.ts` | 可能无需改动（后端接口不变） |

## 三、M2：题目编辑器

### 当前问题

AI 生成的题目偶尔质量差（选项歧义、解析不准确），用户只能接受或忽略，无法修正。

### 设计方案

#### 3.1 入口

在"全部题目"区，每道题旁增加编辑/删除按钮。

```
全部题目（15题）
┌────────────────────────────────────────┐
│ 1. RAG架构的核心组件是什么？            │
│    [编辑] [删除]                       │
│ 2. AI PM 与传统PM的区别？              │
│    [编辑] [删除]                       │
└────────────────────────────────────────┘
```

在 ForgePanel 的"全部题目"区和已攻克列表/稳定卡片列表中均可编辑。

#### 3.2 编辑弹窗

点击编辑 → 内联展开编辑表单：

```
┌────────────────────────────────────────┐
│  编辑题目                              │
│                                        │
│  题目：[................................]│
│                                        │
│  选项 A：[................................]│
│  选项 B：[................................]│
│  选项 C：[................................]│
│  选项 D：[................................]│
│                                        │
│  正确答案：[A] [B] [C] [D]（多选）     │
│                                        │
│  解析：[................................]│
│                                        │
│  [保存] [取消]                         │
└────────────────────────────────────────┘
```

#### 3.3 新增题目

在"全部题目"区底部或顶部加"添加题目"按钮：

点击 → 打开新增表单（结构同上，多了知识点标签选择）

```
┌────────────────────────────────────────┐
│  新增题目                              │
│                                        │
│  知识点标签：[下拉选择]                 │
│  题目：[................................]│
│  选项 ×4...[................................]│
│  正确答案：[A][B][C][D]                │
│  解析：[................................]│
│                                        │
│  [保存] [取消]                         │
└────────────────────────────────────────┘
```

知识点标签从材料已有标签中选择（或自定义输入）。

#### 3.4 删除题目

点击删除 → 确认弹窗 → 删除题目（级联删除关联的 Attempt 和 ReviewCard）。

#### 3.5 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| PUT | `/api/quiz/{question_id}` | 更新题目（题目/选项/答案/解析/标签） |
| POST | `/api/materials/{id}/questions` | 新增题目 |
| DELETE | `/api/quiz/{question_id}` | 删除题目（级联清理） |

#### 3.6 改动文件

| 文件 | 改动 |
|------|------|
| `ForgePanel.tsx` | 全部题目区加编辑/删除按钮，内联编辑表单 |
| `backend/routers/quiz.py` | 新增 `update_question`、`delete_question` 端点 |
| `backend/routers/materials.py` | 新增 `create_question` 端点 |
| `frontend/src/lib/api.ts` | 新增对应 API 调用函数 |

## 四、M3：意图搜索出题（二阶段）

### 目标

用户输入学习意图（如"AI产品经理面试题"），系统自动搜索并出题，无需手动找材料。

### 依赖

- Tavily Search API（项目已有 `tavily-research` skill）
- 内容质量过滤与去重

### 暂定方案

```
输入："大模型微调技术"
    ↓
Tavily 搜索 → 取 Top 3 结果
    ↓
抓取网页内容 → 合并去重 → 分块存储
    ↓
自动出题 → 用户看到时题目已就绪
```

### 暂缓原因

1. 需要额外 API 调用和内容抓取链路
2. 内容质量需要过滤机制
3. v4.3 优先落地无外部依赖的上传简化和题目编辑器
