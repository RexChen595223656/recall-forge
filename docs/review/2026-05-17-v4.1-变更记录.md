# v4.1 变更记录

> 日期：2026-05-17
> 基于 v4.0 的 20+ 项修复和优化

---

## 1. P0 数据正确性

| # | 问题 | 修复 |
|---|------|------|
| 1 | 错题重做判题全错 | 后端比较首字母，前端 `handleWrongSelect` 提取 `charAt(0)` 再传 |
| 2 | 最近锻造显示旧数据 | stats.py 子查询取最新 100 条再正序分组 |

## 2. P1 功能逻辑

| # | 问题 | 修复 |
|---|------|------|
| 3 | 全部题目 toggle 清除标签 | 移除 `setFilterTags([])` |
| 4 | 稳定卡片答案只显示字母 | `startsWith(c.answer)` 匹配完整选项文本 |
| 5 | 异步出题参数硬编码 | generateQuestions 传 mode/difficulty/questionType/tag；后端接收入参 |
| 6 | Config 拆分 (F-25) | QuizConfig(count/feedbackMode/filterTags) + GenConfig(mode/difficulty/questionType/tag) 解耦 |

## 3. P2 代码质量

| # | 问题 | 修复 |
|---|------|------|
| 7 | KnowledgePanel 动态 import | 改为静态导入 `getMaterialStats` |
| 8 | ForgePanel 旧复习死代码 | 删除 review/reviewDone phase、startForge、generateQuizStream |
| 9 | StableCardList 按钮无标签 | 0-3 改为"忘了/印象/记住/简单" |
| 10 | 待加强加载中显示全部 | 未加载完时不显示该材料 |

## 4. 出题流程

| 问题 | 修复 |
|------|------|
| 出题慢（30 题一批 DeepSeek >60s） | max_questions 30→5，单批缩短至 15-30s |
| 轮询不处理 idle 状态 | 增加 idle 分支：0 题则报错 |
| 无进度提示 | 按钮显示"正在出题... (Xs)" |
| 刷新重复触发 | auto-generate 前查 getGenerateStatus，已 generating 则接续 |
| 异常无兜底 | triggerGeneration 加 try/catch + 60 次轮询上限 |
| 上传后无反馈 | UploadModal 提示"正在生成题目..." |
| 生题阻塞答题 | 开始答题按钮去掉 genPending 依赖 |

## 5. 答题流程

| 问题 | 修复 |
|------|------|
| 单选点即提交 | 改为点击选中 + 确认按钮/Enter 提交，与多选一致 |
| 多选题判定失败 | 后端 submit_answer 标准化比较（拆逗号→去空格→排序→拼回） |
| 多选反馈只显示一个选项 | 反馈页/结果页/全部题目解析逗号，多字母匹配 |
| count=2 但出 3 题 | 后端 list_questions 接收 count 参数并 limit |
| count 输入框无法自由输入 | 改用本地 state，blur 时校验提交 |
| count 上限按 chunk 算 | 改为按 total_questions 算 |
| 5 题同批只覆盖 1 chunk | chunk_id 轮询分配而非全取第一个 |
| 题目固定顺序 | list_questions 改为 func.random() 随机排序 |

## 6. UI 修正

| 问题 | 修复 |
|------|------|
| 答题/出题配置混在一起 | 分区标题 + 数量/反馈/开始答题归答题配置，genConfig 归出题配置 |
| 结果页整页滚动条 | 结果列表 max-h 改为 `calc(100vh-22rem)` |
| 0 题进度条显示颜色 | 无题时宽度 0% |
| 0% 进度仍有颜色 | 去掉强行最小 5% |
| 总正确率 + 近期准确率冗余 | 保留一个 |
| 最近锻造命名不直观 | 改为"答题记录" |
| 时间错误（UTC 未转本地） | session date 加 8 小时 |
| 已覆盖"知识点"用词不准 | 改为"段内容" |

## 7. 基础设施

| 问题 | 修复 |
|------|------|
| pytest 破坏 dev server 数据库 | conftest.py 注入测试专用 SQLite 路径 |
| 后端频繁挂掉 | 测试隔离后不再影响开发服 |

## 8. 统计

| 问题 | 修复 |
|------|------|
| 进度条按 attempt 次数算 | 改为按"做对过的题数 / 总题数" |
| 同轮次同一题出现两次 | 同轮次同题去重，保留最新 attempt |
