## 3. 发现问题详情

| # | 严重程度 | 模块 | 问题 | 影响 |
|---|---------|------|------|------|
| 1 | 中 | F5 错题重做 | 正确选项绿色高亮泄露答案 | 错题重做核心体验受损，用户无需思考即可知道答案 |
| 2 | 轻 | F9 标签选题 | 标签单选而非多选，AND 逻辑缺失 | 无法同时筛选多个标签 |
| 3 | 中 | 异常处理 E-06 | 出题失败无 UI 反馈 | 用户不知道出题失败，材料卡片无错误提示 |
| 4 | 轻 | F5/F6 键盘 | ReviewPanel/GlobalReview 无键盘快捷键 | 与 v3.x 答题体验不一致 |

### 修复方案

**问题 1**：ReviewPanel.tsx 和 GlobalReview.tsx 中错题模式移除 `isCorrectOpt` 绿色高亮逻辑，选中答案统一用品牌色（border-brand/50 bg-brand/10 text-brand），选后仅显示 ✓/✗ 文字反馈。

**问题 2**：ForgePanel.tsx 中 `filterTags` 从 `useState<string | null>` 改为 `useState<string[]>`，过滤从单值 `includes` 改为 `filterTags.every(t => q.tags.includes(t))`（AND 逻辑）。

**问题 3**：后端 `routers/materials.py` 新增 `_generation_errors: dict[int, str]`，`run_generation()` 包裹 try/except 捕获异常，`get_generation_status` 返回 error 状态和 message。前端 ForgePanel 新增 `genError` state，空闲态显示红色错误卡片和重试按钮。

**问题 4**：ReviewPanel.tsx 和 GlobalReview.tsx 各新增 keyboard shortcut useEffect，错题模式支持 1-4 选答案、Enter 下一题，SM-2 模式支持 0-3 评分、Enter 确认。
