## 3. API Key 加密存储

| 改动 | 说明 |
|------|------|
| 服务端加密 | Fernet 对称加密，Key 存 SQLite，永不出服务器 |
| 自动检测 | 页面加载时检测 key 状态，未配置自动弹出设置弹窗 |
| 出题拦截 | 无 key 时点击生成/上传，先弹设置窗，拒绝继续 |
| 加密密钥 | 首次启动自动生成 `.fernet_key`，也支持 `FERNET_KEY` 环境变量 |
| 降级策略 | 用户配了用用户的 key，没配用服务器 `.env` 的 key，都没有则报错 |

### 改动文件
- `backend/services/crypto.py` — 新建，Fernet 加解密
- `backend/models/database.py` — 新增 Setting 表
- `backend/routers/settings.py` — 新建，POST/GET /api/settings/key
- `backend/services/quiz_gen.py` — api_key 参数 + 无 key 报错
- `backend/routers/materials.py` — 出题前从 DB 取用户 key
- `backend/routers/quiz.py` — 流式出题同样取用户 key
- `frontend/src/components/ApiKeyModal.tsx` — 新建，设置弹窗
- `frontend/src/components/TopBar.tsx` — 齿轮图标 + 自动检测

## 7. 统计与概念重新设计（2026-05-19）

### 统计指标体系重设计

| 旧指标 | 新指标 | 计算方式 |
|--------|--------|---------|
| 掌握度 | **学习进度** | 做过题数 / 总题数 |
| 正确率 78.6% | **做对 N 题** | 至少做对过 1 次的题数 |
| 已攻克 N 题 | **已攻克 N 题** | 连续 3 次正确，仅 > 0 时低调显示 |

进度条颜色：0-25% 灰、25-65% 黄、65-100% 绿

### SM-2 术语重命名

| 旧名称 | 新名称 |
|--------|--------|
| 间隔复习 | **记忆巩固** |
| 到期 N 题 | **待巩固 N 题** |
| 稳定卡片 | **已掌握** |
| 暂无到期卡片 | **暂无待巩固卡片 / 无需巩固** |

### 改动文件
- `backend/routers/stats.py` — `questions_correct` 字段，进度改为做过数

## 8. 近期修复（2026-05-19）

### 统一批改双重提交 Bug
- **问题**：batch 模式下每题被提交两次（确认时提交首字母，汇聚时提交完整文本），完整文本导致后端判错
- **修复**：batch 模式下不在确认时提交，仅 quizReview 阶段提交一次；userAnswer 存储首字母

### 示例材料三项修复
- **chunk_id**：示例题目分配 chunk_id，显示 20/20 而非 0/20
- **is_example**：MaterialStats 新增 `is_example` 字段，通过 Setting 表追踪，不再用标题匹配
- **已攻克列表**：`<details>` 展开时自动加载题目数据

### 自动标题修复
- **问题**：逐字输入时标题只保留第一个字符（`!title` 在首次设置后不再更新）
- **修复**：用 `titleManEdited` ref 追踪手动编辑，自动标题持续更新直到用户手动改

### API Key 保存体验
- 保存成功自动关闭弹窗
- TopBar 绿色 toast 提示"已保存，加密存储于服务端"，持续 5 秒

### 意图搜索刷新恢复
- 搜索前存 `intent_pending` 到 sessionStorage
- 完成后存 `intent_material_id`
- 页面加载时自动跳回

### 停止生题持久化
- `stoppedMaterials` Set → sessionStorage 存储
- 刷新/切换材料后停止状态不丢失
- KnowledgePanel 加载时自动清理无效材料 ID
- ForgePanel 加载失败时自动清除停止标记

### 生题中开始答题按钮样式
- 有题可用时始终显示正常品牌绿发光样式，不被 genPending 覆盖

- `frontend/src/components/UploadForm.tsx` — 标题修复，占位提示，sessionStorage
