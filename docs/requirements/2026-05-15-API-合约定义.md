# v3.0 锻造台重构 API 合约定义

## 1. 新增接口

### GET /api/materials/{id}/stats

**描述：** 获取单个材料的维度统计（掌握度、历史等）

**请求：** `GET /api/materials/2/stats`

**响应 (200)：**
```json
{
  "material_id": 2,
  "title": "AI产品经理方法论",
  "total_questions": 24,
  "total_attempts": 18,
  "correct_attempts": 14,
  "accuracy": 77.8,
  "mastery": 80,
  "due_reviews": 3,
  "last_activity": "2026-05-15T14:30:00",
  "recent_sessions": [
    {"date": "2026-05-15", "correct": 4, "total": 5, "new_review_cards": 2},
    {"date": "2026-05-14", "correct": 3, "total": 5, "new_review_cards": 1}
  ],
  "tag_distribution": {"AI产品经理": 8, "RAG": 6, "SM-2": 4, "核心能力": 6}
}
```

**响应 (404)：**
```json
{"detail": "材料不存在"}
```

**掌握度计算规则（后端计算）：**
```
mastery = floor(accuracy)  // 基础 = 正确率
if 最近连续答对 >= 3: mastery += 10
if SM-2 最大间隔 > 7天 and 最近评分 >= 2: mastery += 5
if due_reviews > 0: mastery -= 10
clamp(mastery, 0, 100)
```

### GET /api/materials/{id}/stats?include_sessions=true

**描述：** 同上，但 `recent_sessions` 返回最近 10 条而非 5 条

## 2. 修改接口

### POST /api/quiz/generate（无变化）

请求体新增 `feedback_mode` 字段已存在，无需变更。

### GET /api/review/due（无变化）

已支持 `material_id` 和 `show_all` 参数。

## 3. 删除路由

以下路由不再独立存在，功能内嵌到锻造区：

| 旧路由 | 替代方案 |
|--------|---------|
| `/quiz/[id]` | 锻造区 `ForgeActive` 组件 |
| `/review` | 锻造区 `ForgeReview` 组件 |
