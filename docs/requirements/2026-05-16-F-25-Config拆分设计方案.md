# Config 拆分设计方案

> 版本：v4.1（在 v4.0 题目库重构基础上优化）
> 类型：功能拆解 + 交互原型说明（合并 F-06 + F-07）
> 触发：v4.0 PM 验收结束后，发现配置混用问题

---

## 1. 问题诊断

### 1.1 现状

当前 `ForgePanel` 只有一个 config state（`QuizConfig` 类型），包含 6 个字段：

| 字段 | 用途 | 使用者 |
|------|------|--------|
| `count` | 答题数量 | `startQuiz`（题目库模式） |
| `mode` | 出题方式（原文提取/拓展） | `startForge`（旧流式路径，v4.0 已废弃） |
| `difficulty` | 难度（简/中/难） | `startForge`（同上） |
| `questionType` | 题型（单选/多选/混合） | `startForge`（同上） |
| `feedbackMode` | 反馈模式（即时/测验） | `startQuiz` + `startForge` |
| `tag` | 知识点范围 | `startQuiz`（过滤题目） |

### 1.2 核心矛盾

v4.0 引入题目库模式后，答题不再走流式生成，而是直接从已有题目抽取。但 UI 仍把"出题参数"和"答题参数"混在同一个面板里。

**三个具体问题：**

1. **mode/difficulty/questionType 在题目库模式中无意义**：`startQuiz` 走 `listQuestions` 从数据库取题，这三个参数不参与过滤，改了也不生效，用户会困惑
2. **异步出题未传参**：`generateQuestions(materialId)` 没有传 mode/difficulty/questionType，即使 UI 改了这些参数，异步生成也收不到，等于白改
3. **后端缺参**：`POST /api/materials/{id}/generate` 接口不接受 mode/difficulty/questionType，只有 `exclude_covered` 一个可选参数

### 1.3 还有一个隐蔽问题：标签的双重身份

`config.tag` 同时用于两个场景：
- **答题配置**：过滤题目库，只做特定知识点的题
- **出题配置**：生成题目时指定知识点主题

UI 中这是单选的（`useState<string | null>`），但下方"知识标签"展示区又有多选标签系统（`filterTags: string[]`），两套标签逻辑独立运作，用户容易混淆。

---

## 2. 方案设计

### 2.1 拆分原则

```
答题配置（QuizConfig）  → 控制"怎么答题"
出题配置（GenConfig）   → 控制"怎么出题"
```

两条配置完全解耦，各自独立生效。

### 2.2 新接口定义

```typescript
// 答题配置 — 控制当轮答题行为
interface QuizConfig {
  count: number;                    // 题目数量（基于题库实际数量动态步长）
  feedbackMode: "instant" | "batch"; // 即时反馈 / 测验模式
  filterTags: string[];              // 标签筛选（多选，AND 逻辑，空=不限）
}

// 出题配置 — 控制异步出题行为
interface GenConfig {
  mode: "extract" | "expand";       // 原文提取 / 拓展
  difficulty: "easy" | "medium" | "hard"; // 难度
  questionType: "single" | "multi" | "single,multi"; // 题型
  tag?: string;                      // 定向知识点（单选，undefined=不限）
}
```

### 2.3 后端接口补充

`POST /api/materials/{id}/generate` 已接受 `exclude_covered`，需新增可选参数：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| mode | string | "extract" | extract / expand |
| difficulty | string | "medium" | easy / medium / hard |
| question_type | string | "single" | single / multi / single,multi |
| tag | string? | null | 定向知识点 |

后端将这些参数传入 `quiz_gen.py` 的 prompt 构建中（当前 prompt 已有对应占位符，但参数硬编码了默认值）。

## 3. 功能拆解

### 3.1 功能清单

| 模块 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 前端状态 | Config 拆分 | P0 | QuizConfig + GenConfig 两个独立 state |
| 前端状态 | filterTags 并入 QuizConfig | P0 | 删除独立 filterTags state，统一在 quizConfig.filterTags |
| 前端 UI | 答题配置区块 | P0 | count + feedbackMode + filterTags，视觉分组 |
| 前端 UI | 出题配置区块 | P0 | mode + difficulty + questionType + tag，视觉分组 |
| 前端 UI | 按钮归属 | P0 | "开始答题"归答题配置，"补充题目"/"重新出题"归出题配置 |
| 前端逻辑 | startQuiz 使用 quizConfig | P0 | 仅依赖 quizConfig，与 genConfig 无关 |
| 前端逻辑 | supplementQuestions 使用 genConfig | P0 | 将 genConfig 传入 generateQuestions API |
| 后端接口 | generate 接收入参 | P1 | mode/difficulty/question_type/tag → prompt |
| 前端交互 | 出题配置折叠 | P2 | 高频答题场景下出题配置默认折叠，减少干扰 |

### 3.2 状态矩阵

| 功能点 | 正常态 | 空态 | 加载态 | 边界态 |
|--------|--------|------|--------|--------|
| 答题配置 | 显示 count + feedbackMode + filterTags | 无题目时 count 全部置灰，按钮禁用 | — | 题库 0 题：按钮显示"暂无题目" |
| 出题配置 | 显示 mode + difficulty + questionType + tag | tag 列表为空时不显示标签选择 | 出题中按钮显示"出题中..."并禁用 | 已覆盖全部知识点："补充题目"按钮禁用 |
| 开始答题 | 点击即进入答题 | — | — | — |
| 补充题目 | 点击触发异步出题 | — | 按钮 loading + 轮询状态 | 出题失败显示错误卡片+重试 |

## 4. 交互原型说明

### 4.1 空闲态布局（新）

```

```

### 4.2 关键交互

#### 4.2.1 答题配置 — 标签多选

```
点击 [架构] → 高亮选中（品牌色填充）
  ↓
标签不限自动取消（如果之前选中）
全部题目预览区实时过滤
数量选项基于过滤后的题目数动态调整

再点击 [实践] → 同时选中 架构 AND 实践
  ↓
显示同时包含两个标签的题目
```

与下方"题目浏览"区域的标签筛选用的是同一套 `quizConfig.filterTags`。

#### 4.2.2 出题配置 — 标签单选

```
点击 [架构] → 高亮选中
  ↓
下次"补充题目"或"重新出题"时，只针对该知识点生成题目
```

出题标签和答题标签是独立的：
- 答题标签筛选已有题目（多选 AND）
- 出题标签指定生成方向（单选）

#### 4.2.3 开始答题

```
点击 [开始答题]
  ↓
调用 listQuestions(materialId, {
  count: quizConfig.count,
  tags: quizConfig.filterTags.join(","),
  excludeMastered: false
})
  ↓
进入答题阶段（与当前交互一致）
```

不读取 genConfig 任何字段。

#### 4.2.4 补充题目 / 重新出题

```
点击 [补充题目]
  ↓
按钮变为 [出题中...]（disabled）
调用 generateQuestions(materialId, excludeCovered=true, {
  mode: genConfig.mode,
  difficulty: genConfig.difficulty,
  question_type: genConfig.questionType,
  tag: genConfig.tag
})
  ↓ 完成后
按钮恢复，题目库数量增加，统计刷新

点击 [重新出题]
  ↓
同上，但 excludeCovered=false（从全部 chunk 重新生成）
```

#### 4.2.5 出题配置折叠

默认折叠（高频场景是答题不是出题），用户可展开修改。修改 genConfig 不影响已有的题目，只对下次生成生效。

折叠态显示一行概要："出题配置：原文提取 · 中等 · 单选"（可点击展开）。

### 4.3 简化版（取消折叠）

如果认为折叠不够直观（出题配置总共才 3 个选择组 + 1 个标签组），可以不折叠，但答题配置和出题配置之间用明确的分隔线 + 标题区分。

两种方案供选择：
- **方案 A（折叠）**：默认折叠出题配置，降低认知负担
- **方案 B（平铺）**：两个区块都展开，用标题分隔

推荐方案 B，理由：
- 出题配置不复杂（3 行），折叠省的空间有限
- 折叠可能让用户找不到修改入口
- 两个区块清晰并列，用标题和间距区分即可

## 5. 验收清单

### 答题配置

- [ ] **Q-01** quizConfig.count 控制答题数量，与 genConfig 无关
- [ ] **Q-02** quizConfig.feedbackMode 控制反馈模式，即时/测验均可切换
- [ ] **Q-03** quizConfig.filterTags 多选 AND 过滤题目
- [ ] **Q-04** filterTags 为空时从全部题目抽取
- [ ] **Q-05** 选中标签后数量上限自动适配标签题目数
- [ ] **Q-06** 修改答题配置不影响出题配置，反之亦然

### 出题配置

- [ ] **G-01** genConfig.mode 传入 generateQuestions，影响 prompt 的提取/拓展行为
- [ ] **G-02** genConfig.difficulty 传入 generateQuestions，影响题目难度
- [ ] **G-03** genConfig.questionType 传入 generateQuestions，影响生成题型
- [ ] **G-04** genConfig.tag 传入 generateQuestions，限定生成的知识点范围
- [ ] **G-05** 修改出题配置后点"补充题目"，后台收到的参数反映最新配置
- [ ] **G-06** 出题配置默认值与当前一致（extract / medium / single）

### UI

- [ ] **U-01** 答题配置和出题配置有明显视觉分隔（标题 + 分隔线）
- [ ] **U-02** "开始答题"按钮在答题配置区块内
- [ ] **U-03** "补充题目"/"重新出题"按钮在出题配置区块内
- [ ] **U-04** 答题标签多选（视觉可区分选中/未选中）
- [ ] **U-05** 出题标签单选
- [ ] **U-06** 两个标签区视觉上不混淆

### 后端

- [ ] **B-01** `POST /api/materials/{id}/generate` 接受 mode 参数
- [ ] **B-02** 接受 difficulty 参数
- [ ] **B-03** 接受 question_type 参数
- [ ] **B-04** 接受 tag 参数
- [ ] **B-05** 参数正确传入 prompt 构建
- [ ] **B-06** 不传参数时使用默认值（extract / medium / single / null）

### 回归

- [ ] **REG-01** 旧流式生成（startForge）行为不变
- [ ] **REG-02** 已出题目不受 config 修改影响
- [ ] **REG-03** 错题重做和间隔复习入口不变
- [ ] **REG-04** 全部题目浏览区标签筛选仍可用
- [ ] **REG-05** TopBar 全局统计正常刷新
