# 知锻 RecallForge 需求分析文档 v2.0

## 1. 产品定位

**一句话：** AI 驱动的知识内化引擎。把任何信息锻造为长期记忆。

**区别于刷题工具：** 不是做题软件，是"输入材料 → 提取练习 → 间隔重复 → 长期记忆"的完整闭环。面向任何有"真正想内化知识"需求的人。

## 2. 核心闭环

```
材料上传 → 文本分块 → 向量入库（RAG）
    ↓
出题设置 → AI 批量出题 → 答题（即时反馈 / 测验模式）
    ↓
答错自动进入复习队列 → SM-2 间隔重复
    ↓
多次复习 → 间隔逐渐拉长 → 知识内化为长期记忆
```

## 3. 用户角色

| 角色 | 场景 | 核心需求 |
|------|------|---------|
| 学习者 | 阅读了一份报告/课程后想巩固 | 上传材料 → 快速出题 → 验证理解 |
| 备考者 | 系统复习某个知识领域 | 大量出题 → 测验模式 → 错题回顾 |
| 知识工作者 | 日常积累碎片化知识 | 持续导入 → 间隔复习 → 长期巩固 |

## 4. 需求分析（双轨模型）

### 4.1 传统分析轨道

| 角色 | 场景 | 痛点 | 功能 |
|------|------|------|------|
| 学习者 | 读完材料想验证理解 | 不知道记住没有 | 材料上传 + AI 出题 |
| 学习者 | 答错题目 | 下次还会忘 | 自动加入复习队列 |
| 学习者 | 答对题目 | 过段时间也忘了 | SM-2 间隔重复 |
| 备考者 | 想模拟考试 | 即时反馈干扰专注 | 全部做完再看模式 |
| 备考者 | 不同难度练习 | 题目难度不可控 | 简单/中等/困难三档 |
| 知识工作者 | 从网页/PDF导入 | 手动复制粘贴麻烦 | 多格式上传 |

### 4.2 AI 可行性分析

| 维度 | 判断 |
|------|------|
| AI 方案 | LLM 生成题目（DeepSeek V4，禁用 thinking） |
| 数据基础 | 用户上传的材料文本，经 RAG 分块后作为出题素材 |
| 评估标准 | 题目基于材料内容的准确性（人工抽检） |
| 冷启动 | 提供示例材料，10 秒体验完整闭环 |
| 成本 | DeepSeek API，每 5 题约 0.01 元 |

## 7. 设计决策

- **侧边栏导航**：三页应用（知识库/上传/复习），侧边栏比顶栏更 App 化
- **批量出题而非逐题**：一次生成 5 题，做完缓存题 0ms 切换，不逐题等 API
- **不流式展示出题过程**：避免 JSON 原文暴露答案，用动画替代
- **SM-2 而非简单重放**：符合认知科学，避免机械重复
- **答案标准化**：LLM 可能返回"A. 完整选项"，后端保存时只取首字母

## 8. AI 功能验收标准

### 验收卡 1：材料上传 + RAG

| 项目 | 标准 |
|------|------|
| 功能点 | 多格式材料上传并自动分块入库 |
| 输入条件 | 用户提交标题+内容(文本/URL/PDF/MD) |
| 预期输出 | 材料记录创建，文本被分块存入 ChromaDB |
| 准确性标准 | 分块大小约 300 字，overlap 50 字；短于 50 字的内容拒绝 |
| 失败处理 | 格式不支持→提示；文件过大(>10MB)→拒绝；URL超时→提示 |
| 通过条件 | 4 种格式各上传一份，首页列表可见，可跳转刷题 |

### 验收卡 2：AI 批量出题

| 项目 | 标准 |
|------|------|
| 功能点 | 基于材料内容批量生成选择题 |
| 输入条件 | material_id + 五维配置参数 |
| 预期输出 | JSON 数组，每项含 question/options/answer/explanation/tags |
| 准确性标准 | 题目答案基于材料内容可验证；单选 4 选项/多选 5 选项 |
| 失败处理 | chunk 不足→提示上传更多；JSON 解析失败→提示重试；超时 45s→报错 |
| 通过条件 | 5 种配置组合各生成一次，均有题目返回且格式正确 |

### 验收卡 3：答题判定

| 项目 | 标准 |
|------|------|
| 功能点 | 用户选择选项后即时判定对错 |
| 输入条件 | question_id + user_answer(首字母) |
| 预期输出 | is_correct + correct_answer + explanation |
| 准确性标准 | 判定结果与数据库中的 answer 字段一致（标准化后） |
| 失败处理 | 网络失败→前端本地判定兜底 |
| 通过条件 | 正确选项→显示"正确"；错误选项→显示"错误"且展示正确答案 |

### 验收卡 4：SM-2 间隔重复调度

| 项目 | 标准 |
|------|------|
| 功能点 | 根据用户评分计算下次复习时间 |
| 输入条件 | score(0-3) + 当前 ease_factor/interval/repetitions |
| 预期输出 | 更新后的参数 + next_review 时间 |
| 准确性标准 | score<2→间隔重置为1天；score≥2→间隔扩大 |
| 失败处理 | 评分无效→报错"评分必须在 0-3 之间" |
| 通过条件 | 多次评分后验证间隔变化符合 SM-2 公式 |

### 验收卡 5：级联删除

| 项目 | 标准 |
|------|------|
| 功能点 | 删除材料时清理所有关联数据 |
| 输入条件 | material_id |
| 预期输出 | 该材料的 chunks/questions/attempts/reviewCards 全部删除 |
| 失败处理 | 材料不存在→404 |
| 通过条件 | 删除后，复习队列中无该材料相关卡片；统计数字更新 |
