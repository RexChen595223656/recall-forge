## 2. 页面结构详拆

### 2.1 顶部栏（40px，替代原侧边栏）

```
┌─────────────────────────────────────────────────────────┐
│ [R] RecallForge    [今日目标: 3/5题 🔥]    [统计面板] │
└─────────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| Logo | RecallForge + 知锻 |
| 今日目标 | 今日已答题数/目标数（默认5），完成变色 |
| 统计面板 | 悬浮展开：总题数/正确率/连续天数/待复习 |

```
│ └────────────┘ │

```

**材料卡片数据结构（前端计算）：**

```
{
  title: "AI PM方法论",
  totalQuestions: 24,
  correctRate: 0.75,
  mastery: 0.8,         // 掌握度 0-1
  dueReviews: 3,
  lastActivity: "5/15",
  materialId: 2
}
```

**掌握度进度条颜色：**
- ≥80%：绿色（已掌握）
- 40-79%：黄色（进行中）
- 1-39%：红色（薄弱）
- 0%：灰色（未开始）

## 3. 状态矩阵

| 功能点 | 正常态 | 空态 | 加载态 | 错误态 | 边界态 |
|--------|--------|------|--------|--------|--------|
| 知识库列表 | 材料卡片+掌握度 | 引导文案+示例按钮 | 骨架卡片×3 | 重试按钮 | 超过20条→滚动 |
| 锻造区空闲 | 材料统计+历史 | 提示"选择左侧材料" | — | — | 材料被删除→提示不存在 |
| 锻造区生成 | 批量返回题目 | — | 圆点动画+取消 | 超时→重试 | 该材料全部chunk已用过→提示无新题 |
| 锻造区答题 | 选项可点击 | — | — | 本地判定兜底 | 多选→勾选后确认 |
| 锻造区反馈 | 正确/错误+解析 | — | — | — | 最后一题→"完成"替代"下一题" |
| 锻造区结果 | 统计+逐题回顾 | — | — | — | 全部正确→无加入复习 |
| 复习模式 | SM-2评分卡片 | 全部已掌握→提示 | — | — | 无到期卡片→提示"暂无待复习" |

## 5. 数据流

```
MaterialCard.onClick(materialId)
    → page state: selectedMaterial = materialId
    → ForgePanel 切换到空闲态，加载该材料统计
        → GET /api/materials/{id}/stats (新增API)
            → { totalQuestions, correctRate, mastery, dueReviews, recentSessions }
    → 用户点击"开始锻造"
        → ForgePanel 切换到设置态(折叠)
        → 用户确认设置 → ForgePanel 切换到锻造中
            → POST /api/quiz/generate (现有API)
            → 批量出题
            → 答题流程 (现有逻辑)
            → 结果态 → 更新材料统计
    → 用户点击"仅复习薄弱点"
        → ForgePanel 切换到复习模式
            → GET /api/review/due?material_id={id}
            → SM-2 评分流程 (现有逻辑)
```

## 6. 路由变更

| `/quiz/[id]` | 删除 | 锻造区内嵌 |
| `/review` | 删除 | 锻造区内嵌 |

| — | `/?material=2` | 新增：URL query 指定材料 |
