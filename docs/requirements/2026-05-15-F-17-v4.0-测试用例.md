## 测试数据集

### 材料 A：短文（1 chunk）

```
标题：短文测试
内容：AI产品经理需要掌握大模型能力边界、RAG架构、Prompt工程化、评估体系和数据飞轮五项核心能力。这些能力共同构成AI PM区别于传统PM的竞争力。
```

预期：1 chunk → 生成 3 题。

### 材料 B：中长文（3-4 chunks）

```
标题：RAG架构详解
内容：检索增强生成（RAG）是当前大模型应用的核心架构。它通过将外部知识库与模型生成能力结合，有效减少幻觉问题。

RAG的工作流程分为三个阶段。首先是检索阶段，系统将用户查询向量化后在向量数据库中搜索最相关的文档片段。向量数据库如ChromaDB、Pinecone等存储文档的embedding向量。

其次是增强阶段，检索到的相关文档片段与原始查询拼接形成增强prompt。这个阶段的难点在于如何选择最相关且不冗余的片段。

最后是生成阶段，大模型基于增强prompt生成回答。RAG的优势在于不需要微调模型就能接入最新知识，且回答可追溯来源。

RAG的典型应用场景包括企业知识库问答、客服系统、法律文书检索等。在实际部署中还需要考虑文档分块策略、embedding模型选择、检索精度与召回率的平衡。
```

预期：3-4 chunks → 生成 9-12 题。

## TC6：间隔复习

### TC6.1 入口与列表

| 步骤 | 操作 | 预期 |
|------|------|------|
| 前置 | 有 2 张到期卡片（next_review ≤ now） | |
| 1 | 材料页显示"间隔复习（2题到期）" | 入口可见 |
| 2 | 点击进入 | 显示第 1 道题 |
| 3 | 头部 | "间隔复习" + "复习 N 次 · 连续正确 M 次" |

### TC6.2 复习评分

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 题目显示，选项不可点击 | 初始状态 |
| 2 | 点击"回忆一下，点击查看答案" | 正确答案高亮 |
| 3 | 点"记住了"(2) | next_review 延后，repetitions +1 |
| 4 | 点"忘记了"(0) | next_review = 明天，repetitions = 0 |
| 5 | total_reviews | 每次评分后 +1，永远递增 |

### TC6.3 毕业判定

| 条件 | 预期 |
|------|------|
| interval=25, reps=2 | 不毕业（reps < 3） |
| interval=18, reps=4 | 不毕业（interval < 21） |
| interval=25, reps=3 | **毕业** → is_stable=true，提示"已稳定" |
| interval=35, reps=5 | **毕业** → is_stable=true |

### TC6.4 手动复习重新激活

| 步骤 | 操作 | 预期 |
|------|------|------|
| 前置 | 某卡片 is_stable=true | |
| 1 | 进入"稳定卡片"，找到该卡片 | 可查看 |
| 2 | 点击"手动复习" | 进入复习界面，is_stable=false，interval=1 |
| 3 | 评分 0 | next_review=明天，回到活跃队列 |
