## Bug #1：错题重做模式泄露正确答案

**严重程度**：中
**影响功能**：F5 错题重做
**不通过项**：F5-05、F5-06

### 根因

`ReviewPanel.tsx` 和 `GlobalReview.tsx` 中，错题模式下选项渲染复用了 SM-2 模式的 `isCorrectOpt` 判断逻辑。该逻辑在用户提交答案后，通过 `opt.startsWith(item.answer)` 将正确选项标为绿色高亮。

错题重做的核心规则是"不显示正确答案"，用户只能通过 ✓/✗ 文字反馈知道自己答对还是答错，不应看到哪个选项是正确的。

### 修复

**ReviewPanel.tsx** 和 **GlobalReview.tsx**：

1. 移除错题模式下选项渲染中的 `isCorrectOpt` 绿色高亮分支
2. 选中答案统一用品牌色：`border-brand/50 bg-brand/10 text-brand`
3. 未选中答案置灰 `opacity-40`
4. 选后仅通过下方文字显示 ✓ 正确 / ✗ 错误

修改位置：
- `ReviewPanel.tsx:165-175` — 选项 class 逻辑分支
- `GlobalReview.tsx:119-128` — 选项 class 逻辑分支

### 修复后验证

- 错题重做模式下，选前所有选项外观一致
- 选后用户选择的选项以品牌色高亮，正确选项无绿色标注
- 仅显示 ✓/✗ 文字反馈

## Bug #2：标签筛选仅支持单选

**严重程度**：轻
**影响功能**：F9 标签选题
**不通过项**：F9-01、F9-07

### 根因

`ForgePanel.tsx` 中 `filterTags` 状态定义为 `useState<string | null>(null)`，每次点击标签直接替换值，无法同时选中多个标签。

功能拆解中明确要求：标签云可多选，选中多个标签时 AND 逻辑过滤（题目需包含所有选中标签）。

### 修复

**ForgePanel.tsx**：

1. `filterTags` 改为 `useState<string[]>([])`
2. 过滤逻辑从 `filterTags && q.tags.includes(filterTags)` 改为 `filterTags.every(t => q.tags.includes(t))`
3. 标签按钮 `onClick` 改为 toggle 操作：
```typescript
setFilterTags(prev => prev.includes(tag) ? prev.filter(t => t !== tag) : [...prev, tag]);
```

### 修复后验证

- 可同时选中多个标签，每个选中的标签高亮
- 题目列表按 AND 逻辑过滤，只显示包含所有选中标签的题目
- 取消全部选中后恢复显示全部题目

## Bug #3：出题失败无 UI 反馈

**严重程度**：中
**影响功能**：异常处理 E-06
**不通过项**：E-06

### 根因

后端 `routers/materials.py` 中 `run_generation()` 在线程中执行，异常未捕获并向外传递。`get_generation_status` 接口只返回 idle/generating/ready 三种状态，无法感知失败。

前端仅轮询 status 字段，没有 error 状态处理分支。

### 修复

**后端** `routers/materials.py`：

1. 新增 `_generation_errors: dict[int, str] = {}` 存储失败信息
2. `run_generation()` 包裹 try/except，捕获异常写入 `_generation_errors[material_id]`
3. `get_generation_status` 检查 `_generation_errors`，有则返回 `{"status": "error", "message": "..."}` 并清除

**前端** `ForgePanel.tsx`：

1. 新增 `genError` state
2. 轮询 status 时检查 `status === "error"`，设置 `genError = data.message`
3. 空闲态渲染红色错误卡片：
```tsx
{genError && (
  <div className="text-red-400 text-xs bg-red-400/5 px-3 py-2 rounded-lg space-y-2">
    <p>{genError}</p>
    <button onClick={retry}>点击重试</button>
  </div>
)}
```

**前端** `api.ts`：

- `GenerateStatus` 接口新增 `message?: string` 字段

### 修复后验证

- 模拟出题 API 失败后，材料卡片显示错误文案
- 空闲态显示红色错误卡片 + 重试按钮
- 点击重试重新调用出题 API

## Bug #4：ReviewPanel 和 GlobalReview 缺少键盘快捷键

**严重程度**：轻
**影响功能**：F5 错题重做、F6 间隔复习
**不通过项**：F5-10

### 根因

v4.0 重构时，ReviewPanel 和 GlobalReview 是独立新建的组件，未从原 ForgePanel 答题逻辑中继承键盘快捷键支持。而 v3.x ForgePanel 答题模式已有完善的键盘快捷键（1-4 选选项、Enter 下一题、0-3 SM-2 评分）。

### 修复

**ReviewPanel.tsx** 和 **GlobalReview.tsx** 各新增 keyboard shortcut useEffect：

```typescript
useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if (loading || finished || items.length === 0) return;
    const item = items[idx];
    if (!item) return;
    if (mode === "wrong") {
      const qItem = item as QuestionItem;
      if (!submitted) {
        const keyIdx = parseInt(e.key) - 1;
        if (keyIdx >= 0 && keyIdx < qItem.options.length) handleWrongSelect(qItem.options[keyIdx]);
      } else if (e.key === "Enter") {
        advance();
      }
    } else {
      if (!showAnswer) {
        if (e.key === "Enter") setShowAnswer(true);
      } else {
        const keyIdx = parseInt(e.key);
        if (keyIdx >= 0 && keyIdx <= 3) handleSm2Score(keyIdx);
        else if (e.key === "Enter") advance();
      }
    }
  };
  window.addEventListener("keydown", handler);
  return () => window.removeEventListener("keydown", handler);
}, [loading, finished, items, idx, mode, submitted, showAnswer]);
```

### 修复后验证

- 错题模式：按 1-4 选择对应选项，按 Enter 进入下一题
- SM-2 模式：按 0-3 评分，按 Enter 跳过/确认
- 与其他组件键盘事件无冲突

## 总结

| Bug | 根因类型 | 影响模块 | 修复文件数 |
|-----|---------|---------|-----------|
| #1 答案泄露 | 状态分支遗漏，复用错误逻辑 | ReviewPanel, GlobalReview | 2 |
| #2 单选标签 | 数据类型选择错误，`null` vs `[]` | ForgePanel | 1 |
| #3 错误无声 | 异常未传递到 UI 层 | materials.py, ForgePanel, api.ts | 3 |
| #4 快捷键缺失 | 新组件未继承旧交互 | ReviewPanel, GlobalReview | 2 |

**教训**：
- 新功能不要复用`isCorrectOpt`逻辑——它的"正确答案高亮"语义与错题重做的"不暴露答案"规则冲突，导致泄露。应拆分独立的条件分支
- 异步线程异常必须显式捕获并向上传递，fire-and-forget 不等于 forget the error
- 新组件要对照交互原型说明中的全量交互节点检查，不能遗漏键盘快捷键这类隐式需求
