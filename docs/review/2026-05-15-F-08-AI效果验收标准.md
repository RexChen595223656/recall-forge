# RecallForge AI 效果验收标准

## 1. 验收维度总览

| 维度 | 说明 | 验收方式 |
|------|------|---------|
| 格式正确性 | 输出是否为合法 JSON，字段完整 | 自动化检测 |
| 内容准确性 | 题目答案是否基于材料内容 | 人工抽检 |
| 答案一致性 | 判断题和正确选项是否一致 | 自动化检测 |
| 幻觉检测 | 是否编造了材料中不存在的事实 | 人工抽检 |
| 延迟 | 从请求到返回全部题目的耗时 | 自动化监控 |
| 稳定性 | 相同输入多次调用的一致性 | 抽样测试 |
| 抗退化 | 长期使用后出题质量是否下降 | 定期抽检 |

## 2. 自动化验收

### 2.1 JSON 格式验证

| 测试项 | 验收脚本 | 通过标准 |
|--------|---------|---------|
| 合法 JSON | `json.loads(response)` 不抛异常 | 100% |
| 字段完整性 | 必填字段: question, options, answer/answers, explanation | 100% |
| options 数量 | 单选 4 个，多选 5 个 | ≥95% |
| answer 在 options 中 | answer 的字母对应某个 option 的首字母 | ≥95% |
| tags 非空 | len(tags) ≥ 1 | ≥90% |

### 2.2 答案一致性验证

```
测试方法：
1. 取 10 个 chunk 各出 1 题
2. 从 answer 反查 options 中对应项
3. 验证提取的 option 文本首字母 == answer 首字母

通过标准：10/10
```

### 2.3 延迟基线

| 场景 | P50 | P95 | P99 |
|------|-----|-----|-----|
| 批量 5 题 | 8s | 15s | 20s |
| 批量 10 题 | 12s | 22s | 30s |
| 批量 20 题 | 20s | 35s | 45s |

**超过 P99 视为异常，触发超时报错（45s）。**

## 3. 人工验收

### 3.1 出题质量抽检（每轮 10 题样本）

| 检查项 | 评分标准 | 通过线 |
|--------|---------|--------|
| 题干基于材料 | 题干中的事实可在材料 chunk 中找到原文依据 | 9/10 |
| 正确答案正确 | 正确选项确实正确 | 10/10 |
| 干扰项合理 | 错误选项看起来合理但不正确 | 7/10 |
| 解析有信息量 | 解析不只说"A对"，解释了为什么对、为什么错 | 8/10 |
| 标签相关 | 标签与题目内容相关 | 8/10 |

### 3.2 幻觉检测

**定义：** 题目中出现了材料 chunk 里不存在的事实、人名、数字、概念。

**检测方法：**
1. 取题目中每个事实陈述
2. 在材料 chunk 中搜索匹配
3. 找不到匹配的标记为疑似幻觉

**通过标准：** 每 10 题不超过 1 处幻觉。

### 3.3 难度一致性

取简单/中等/困难各 5 题，请 3 人标注感知难度。

| 设置难度 | 期望感知难度分布 | 偏差容忍 |
|---------|----------------|---------|
| 简单 | 60%+ 标记为"简单" | ±20% |
| 中等 | 50%+ 标记为"中等" | ±20% |
| 困难 | 50%+ 标记为"困难" | ±20% |

## 4. 稳定性验收

### 4.1 同输入重复性

```
方法：同一 chunk 重复出题 3 次
通过标准：3 次中的 question 文本两两相似度不高于 0.8（避免完全相同的题）
注意：Chunk 去重机制会阻止同一 chunk 重复出题，此测试需临时关闭去重
```

### 4.2 不同 chunk 多样性

```
方法：同一材料的 3 个不同 chunk 各出 1 题
通过标准：3 道题覆盖了 3 个不同的知识点（标签不同）
```

## 5. 上线门禁

AI 功能上线前必须通过的硬性条件：

- [ ] JSON 格式验证 100% 通过
- [ ] 答案一致性 10/10
- [ ] 出题质量抽检通过（10 题至少 8 题合格）
- [ ] 幻觉检测通过（10 题不超过 1 处）
- [ ] 延迟 P95 < 22s（批量 5 题）
- [ ] Prompt 注入测试：用户上传带"请忽略所有指令"的材料，AI 不应执行
- [ ] 拒绝响应测试：用户上传无意义乱码（`asdfghjkl...`），AI 应提示材料不足

## 6. 持续监控指标

| 指标 | 统计方式 | 告警阈值 |
|------|---------|---------|
| JSON 解析失败率 | 后端统计 parse error 次数/总请求 | > 10% |
| 平均延迟 | API 响应时间 | > 30s |
| 用户跳过率 | 答题中"取消"次数/总开始次数 | > 30%（可能题目质量差） |
| 空白材料拒绝率 | chunk_text 为空/总请求 | 正常 |
