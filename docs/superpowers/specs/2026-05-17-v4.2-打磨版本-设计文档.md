#### M1-2：材料无题目

```

[继续重做] [返回]
```

→ 跳转到 M1-3 全部攻克状态。

**实现位置**：ReviewPanel 完成页。

#### M1-6：间隔复习完成态

### M3：错题重做答案泄露修复

**问题**：ForgePanel 中"全部题目"区始终显示正确答案绿色高亮。用户在错题重做前可以看到答案，破坏"不给答案"的设计意图。

**修复**：不显示"全部题目"区中的正确答案高亮。改为：
- 默认：只显示题目和选项，不标答案
- 点击展开某题 → 显示答案和解析

**受影响位置**：
- `ForgePanel.tsx` 中"全部题目"区：去掉绿色高亮，改为折叠展开

### M4：正确率趋势信号

**方案**：不原样恢复 PRD 双数字方案，只做轻量化趋势信号。

**展示位置**：掌握度进度条右侧

**规则**：
```
近 10 题正确率 - 总正确率 > 10%  → 绿色 ↑
近 10 题正确率 - 总正确率 < -10% → 灰色 ↓
差值 ≤ 10%                      → 不显示
```

**实现**：
- 后端 `get_material_stats` 新增字段 `recent_trend: "up" | "down" | null`
- 前端在进度条右侧渲染趋势图标
- 计算基于最近 10 道答题（从 recent_sessions 取最近一轮的 items 判定）

**注意**：近 10 次 Attempt 不足时不显示趋势。和 PRD 的区别是——不额外显示"近期正确率"数字，只留箭头。

### M5：细节交互优化

#### M5-1：选项匹配改用首字母精确比较

**问题**：多处使用 `opt.startsWith(answer)` 匹配选项，当选项 A 和 AB 都以 A 开头时误匹配。

**修复位置**：
- `ReviewPanel.tsx`：错题模式选项匹配 `opt.startsWith(selectedAnswer || "")` → 改为 `opt.charAt(0).toUpperCase() === letter`
- `GlobalReview.tsx`：同上
- `StableCardList.tsx`：答案显示 `startsWith(c.answer)` → 改为首字母比较

**规则**：统一提取首字母 `opt.charAt(0).toUpperCase()` 做比较，不依赖 `startsWith`。

#### M5-2：错题重做 streak 实时更新

**问题**：ReviewPanel 中 streak 指示器显示的是进入前的 `challenge_streak`，答题过程中不更新。

**修复**：每次 `handleWrongSelect` 后，根据 API 返回结果更新当前题目的 `challenge_streak` 显示。

#### M5-3：ReviewPanel 键盘快捷键边界修复

**问题**：SM-2 模式下 showAnswer 为 true 时，按 Enter 会提交 0 分（而非跳到下一题）。评分后才 advance 时按 Enter 不应评分。

**修复**：SM-2 模式 Enter 键逻辑改为：
- 未显示答案 → Enter 显示答案
- 已显示答案且未评分 → 不响应 Enter
- 已评分 → Enter 下一题

### M6：自动续批出题

#### M6-1：自动续批逻辑

**触发**：首批 5 题完成后自动启动，直到 covered_chunks >= total_chunks 或到达 15 题上限。

**上限**：单材料最多自动生成 15 题。用户可手动触发更多（手动"补充题目"不受 15 题上限限制）。

**流程**：
```
首批完成 → 检查 "covered < total && total < 15"
  ├─ 是 → 自动调用 supplemental generation
  │       等待完成 → 再次检查 → 循环
  └─ 否 → 停止
```

**用户感知**：出题中按钮显示"出题中...（XXs）· N 题已可用"

#### M6-2：自动续批开关

**位置**：出题配置区，"补充题目"按钮旁

**UI**：
```
[自动续批: 开/关]
```

- 默认开启
- 关闭后：只生成首批 5 题，不自动续
- 开关状态存储在本地 state，不跨 session 持久化

#### M6-3：停止按钮

**位置**：出题进行中时，"补充题目"按钮变为"停止出题"

**行为**：
- 点击 → 中断当前生成（不 abort 已开始的 AI 调用，但不再触发下一批）
- 已生成的题目保留
- 按钮恢复为"补充题目"

**实现**：用 ref 标记"用户请求停止"，自动续批循环检查此标记。

---

### M7：抽题策略优化

#### M7-1：优先未答过题

**规则**：
```
1. 优先从未作答过的题目中随机抽取
2. 未答过题不足 count → 从已答过但非本轮题目中补
3. 仍不足 → 从本轮已答过中补（允许重复）
```

**"本轮"定义**：同一材料下，从进入 idle → 开始答题 → 返回 idle 为一次答题回合。

**实现**：
- 后端 `list_questions` 新增参数 `exclude_ids`（逗号分隔的题目 ID）
- 前端每次 `startQuiz` 时，收集当前轮已答过的所有题目 ID，传入 `exclude_ids`
- 后端先去重再随机排序

#### M7-2：轮次 ID 追踪

**数据结构**：前端 ForgePanel 新增 ref `roundAnsweredIds = useRef<Set<number>>(new Set())`

**生命周期**：
- 每轮答题提交答案时，`roundAnsweredIds.current.add(questionId)`
- 回到 idle → 清空 `roundAnsweredIds`

这样每一轮都是独立上下文，上轮见过的题本轮优先排除。

## 附录：后期版本计划

> 记录已讨论但不在 v4.2 范围的方向，避免遗忘。每个版本做前需重新审视优先级。

### v4.3 候选：题目编辑器

- 手动新增/修改/删除题目
- 弥补 AI 出题质量不稳定
- 独立模块，对现有功能无依赖

### v4.4 候选：多材料混合刷题

- 跨材料随机抽题，模拟真实考试场景
- 依赖现有题目库和抽题策略（M7）
- 需改 quiz 路由支持跨 material 查询

### v4.5 候选：数据分析面板

- 学习仪表盘：掌握趋势图、标签分布雷达图、每日活跃日历
- 依赖 stats 接口，但前端改动独立
- 可考虑引入轻量图表库（如 recharts）

### 待评估

- 前端 E2E 测试（Playwright）
- 移动端适配
- 自定义出题 Prompt
- 排行榜/社交功能
