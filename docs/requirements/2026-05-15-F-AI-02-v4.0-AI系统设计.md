# v4.0 AI 系统设计 — 异步出题架构

## 一、架构变更

```
v3.x（同步 SSE）:
  用户点击锻造 → SSE 流式 → 等待 60-90s → 返回题目

v4.0（异步 fire-and-forget）:
  上传材料 → 前端 fire POST /generate → 后端异步出题
  用户继续浏览 → 轮询状态 → 题目就绪后通知
```

### 为什么改用异步

| 维度 | v3.x 同步 | v4.0 异步 |
|------|----------|----------|
| 用户等待 | 60-90s 阻塞 | 0s（题目提前生成） |
| 生成时机 | 每次答题前 | 上传时一次 |
| 失败处理 | 显示错误，重试 | 卡片提示失败，可重试 |
| 题目复用 | 答完就没了 | 存在 DB 随时用 |

## 二、异步出题接口

### 端点

```
POST /api/materials/{material_id}/generate
```

### 请求

```json
{
  "exclude_covered": true
}
```

`exclude_covered=true`：仅从 uncovered chunk 出题（补充题目时用）。
默认 `false`：从所有 chunk 出题（首次出题时用）。

### 行为

1. 接口立即返回 `{"status": "generating"}`（不等待 LLM）
2. 后台启动异步任务调用 LLM
3. LLM 返回后保存题目到 DB
4. 前端通过 `GET /api/materials/{material_id}/status` 轮询状态

### 状态轮询

```
GET /api/materials/{id}/status
→ {
    "status": "idle" | "ready" | "error",
    "question_count": 0,                         // idle 时
    "max_questions": 9                           // 预期出题数
  }
```

注：Thread 方案不支持中间进度推送，前端通过轮询 `status` 是否变为 `ready` 来判断完成。

### 异步实现方案

选择 `threading.Thread`：
- 零额外依赖
- 适合当前项目规模（单用户本地使用）
- 局限：进程重启会丢失正在进行的生成任务（可接受——前端轮询超时后显示"出题失败，点击重试"即可）
- 不使用 FastAPI BackgroundTasks：因为 BackgroundTasks 无法在任务执行期间更新进度状态

### 出题数量公式

```python
max_questions = min(total_chunks * 3, 30)
if max_questions < 1:
    max_questions = 1
```

### 题目保存流程

每道题关联一个 chunk_id：

```python
# 从 chunks 中轮询分配 chunk_id
for i, question in enumerate(questions):
    chunk_id = chunk_ids[i % len(chunk_ids)]
    Question(
        material_id=material_id,
```

## 三、Prompt 设计

与 v3.x Prompt 相同，仅调整出题数量。关键参数保持不变：
- `"thinking": {"type": "disabled"}` — 必须禁用，否则 DeepSeek 返回 thinking_delta 导致解析失败
- `"max_tokens": 4096` — 30 题比 5 题需要更多 token
- `"stream": false` — 异步出题用非流式，简化解析

其余 Prompt 结构不变（模式指令、难度指令、题型指令、标签指令、格式要求）。v4.0 中"模式"和"难度"保留在 Prompt 中，因为首次出题用默认值（extract + medium），补充题目时沿用相同设置。

## 四、题目补充逻辑

```
补充条件：covered_chunks < total_chunks
补充数量：min((total_chunks - covered_chunks) * 3, 30)
出题范围：仅 uncovered chunks
```

### uncovered chunks 查询

```python
def get_uncovered_chunks(material_id: int) -> list:
    # 已使用的 chunk_id
    used_ids = db.query(Question.chunk_id).filter(
        Question.material_id == material_id,
        Question.chunk_id != ""
    ).all()
    used = set(r[0] for r in used_ids)

    # 全部 chunk（ChromaDB get 返回格式: {ids, documents, metadatas}）
    result = collection.get(where={"material_id": material_id})
    chunks = []
    if result and result.get("documents"):
        for i, doc in enumerate(result["documents"]):
            meta = result["metadatas"][i] if result.get("metadatas") else {}
            if meta.get("chunk_oid", "") not in used:
                chunks.append({"content": doc, "metadata": meta})

    return chunks
```

## 六、错题攻克状态

### 判定逻辑（后端）

`POST /api/quiz/submit` 端点每次提交时判定：

```
if user_answer == correct_answer:
    question.challenge_streak += 1
    if question.challenge_streak >= 3:
        question.mastered = True
else:
    question.challenge_streak = 0
    question.mastered = False
```

前端从响应中读取 `challenge_streak` 和 `mastered`，展示 streak 进度和攻克动画。

### 数据模型

Question 表：

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| challenge_streak | int | 0 | 连续做对次数 |
| mastered | bool | false | 是否已攻克 |

## 七、ReviewCard 稳定判定

### 毕业逻辑

```python
def record_review(...):
    result = calculate_sm2(score, ...)
    
    # 更新 SM-2 状态
    card.repetitions = result["repetitions"]
    card.interval_days = result["interval_days"]
    card.total_reviews += 1
    
    # 毕业判定：间隔 > 21 天且连续成功 >= 3 次
    if card.interval_days > 21 and card.repetitions >= 3:
        card.is_stable = True
    
    card.next_review = result["next_review"]
```

### 手动复习后重新激活

稳定卡片可通过 `POST /api/review/record` 手动复习，不执行 SM-2 计算，直接走分支：

```python
def record_review(...):
    if card.is_stable:
        if score < 2:
            # 记忆已弱化，重新进入复习循环
            card.is_stable = False
            card.interval_days = 1
            card.repetitions = 0
        # score >= 2: 保持稳定，仅更新 last_review
        card.last_review = datetime.utcnow()
        db.commit()
        return {"ok": True, "interval_days": card.interval_days, "is_stable": card.is_stable}

    # 非稳定卡片：正常 SM-2 流程
    result = calculate_sm2(score, ...)
    ...
```
