# 知锻(RecallForge) — AI 可行性评估

## 1. 需求描述

从用户上传的学习材料中自动生成高质量选择题，并基于答题记录实现间隔重复复习。核心 AI 能力：材料理解 + 题目生成 + 答案判定。

## 2. AI 方案判断

- [x] 适合 AI 解决 → RAG + LLM 生成方案
- [ ] 规则/传统方案更好
- [ ] 混合方案

**判断依据：** 从非结构化文本中理解内容、提炼考点、生成干扰项——这些需要语言理解和生成能力，传统规则无法做到。答案判定是确定性比较，可用规则解决。

## 3. 推荐方案详情

- **架构选择：** RAG（检索增强生成）
  - 用户材料 → 分块 → Embedding → ChromaDB
  - 出题时：检索 Top-K 相关 chunk → Prompt 模板 → LLM 生成
- **模型选型：** DeepSeek (deepseek-v4-pro)
  - 理由：中文能力强，成本低，已配置 Anthropic 兼容端点
  - 备选：如需更强推理可用 Claude Opus 4
- **数据需求：** 无需预训练数据。用户上传的材料即为数据源。Embedding 模型候选：text-embedding-3-small 或 DeepSeek Embedding API
- **成本估算：**
  - DeepSeek API：约 1-2 元/万字（题目生成）
  - Embedding：约 0.1 元/万字
  - ChromaDB：免费（嵌入式模式）
  - 月度预估：个人使用 < 50 元
- **延迟预期：**
  - Embedding：5-10s / 万字材料（实际使用 ChromaDB 内置 embedding）
  - 题目生成：P50 9s（DeepSeek V4，thinking disabled），禁用 thinking 前约 18s

## 4. 风险与 mitigation

- **能力风险：** LLM 可能生成与材料无关的题目或编造事实 → RAG 绑定 chunk，Prompt 强制要求引用来源
- **数据风险：** 用户上传内容可能含敏感信息 → 所有数据本地存储，不上传第三方
- **成本风险：** 高频使用可能导致 API 费用上升 → 缓存已生成题目，加入每日额度提示
- **替代方案：** 如果 DeepSeek 不可用，可切换至 Claude API（Anthropic 兼容端点）

## 5. 结论与建议

**已实施。** RAG + LLM 方案已验证可行。实际部署使用 ChromaDB 嵌入式模式，DeepSeek V4（thinking disabled）实现约 9s 的题目生成延迟。五维配置 + 批量出题 + SM-2 间隔重复 + 级联删除均已实现。详见 PRD v2 和功能拆解 v2。
