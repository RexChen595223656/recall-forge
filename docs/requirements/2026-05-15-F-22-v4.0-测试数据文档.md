### 材料 C：中文（3-4 chunks → 9-12 题）

```text
检索增强生成（RAG）是当前大模型应用的核心架构。它通过将外部知识库与模型生成能力结合，有效减少幻觉问题，让模型能够基于最新、最准确的信息回答问题。RAG的出现改变了传统AI产品对模型训练的依赖。

RAG的工作流程分为三个关键阶段。首先是检索阶段，系统将用户查询向量化后在向量数据库中搜索最相关的文档片段。向量数据库如ChromaDB和Pinecone存储文档的embedding向量，通过余弦相似度计算找到最相关的内容。

其次是增强阶段，检索到的相关文档片段与原始查询拼接形成增强prompt。这个阶段的难点在于如何选择最相关且不冗余的片段，以及如何处理检索结果为空的情况。

最后是生成阶段，大模型基于增强prompt生成回答。RAG的优势在于不需要微调模型就能接入最新知识，且回答可追溯来源。但RAG也有局限，比如检索精度不足时可能引入噪音，需要精心设计分块策略和检索参数。

RAG的典型应用场景包括企业知识库问答、智能客服系统、法律文书检索、医疗文献查询等。在实际部署中还需要考虑文档分块策略、embedding模型选择、检索精度与召回率的平衡、以及成本控制。对于AI产品经理来说，理解RAG的运作原理和适用边界是设计可靠AI产品的必要条件。
```

~600 字，5 个段落，预期 4-5 chunks → 12-15 题。

## 三、测试数据准备脚本

```python
# tests/utils/seed_data.py
"""Run before E2E tests to create known test data"""
import requests
import time

API = "http://localhost:8000/api"

def create_material(title, content):
    resp = requests.post(f"{API}/materials", data={"title": title, "content": content})
    resp.raise_for_status()
    data = resp.json()
    mid = data["id"]
    # Trigger async generation
    requests.post(f"{API}/materials/{mid}/generate")
    # Wait for completion
    for _ in range(30):
        status = requests.get(f"{API}/materials/{mid}/status").json()
        if status["status"] in ("ready", "error"):
            break
        time.sleep(2)
    return mid

def seed_basic():
    """最小测试集：1 个材料 + 3 道题"""
    short_content = "AI产品经理的核心能力包括理解大模型能力边界、设计RAG架构、工程化Prompt系统、建立多层评估体系和构建数据飞轮五项。这些能力与传统PM的需求分析、用户体验设计能力互补。掌握这些能力的PM能更好地与技术团队协作设计AI产品。"
    return create_material("基础测试", short_content)

def seed_full():
    """完整测试集：3 个材料，含已有答题记录"""
    # 材料 1：标准
    mid1 = seed_basic()
    # 材料 2：多标签
    medium_content = "..."  # 使用材料 C 的内容
    mid2 = create_material("RAG 详解", medium_content)
    # 材料 3：长文
    long_content = "..."  # 使用材料 D 的内容
    mid3 = create_material("AI PM 完整指南", long_content)

if __name__ == "__main__":
```
