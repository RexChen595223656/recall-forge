# v4.0 功能拆解 — 状态矩阵 + 逻辑流程

---

## F1：上传自动出题

### 状态矩阵

| 状态 | 触发条件 | 材料卡片显示 | 题目区显示 | 可操作 |
|------|---------|-------------|-----------|--------|
| idle | 材料刚上传 | "0 题" | — | 浏览原文 |
| generating | 后端开始出题 | "出题中..."（文案+动画） | 加载动画 | 浏览原文、查看已有题 |
| ready | 出题完成 | "N 题可用" | 全部题目 + 开始答题 | 全部操作 |
| error | 出题失败 | "出题失败" | 重试按钮 | 重试、浏览原文 |

### 逻辑流程

```
用户上传材料
  ↓
前端调用 POST /api/materials → 返回 material {id, chunk_count}
  ↓
前端调用 POST /api/materials/{id}/generate（不 await，fire-and-forget）
  ↓ 同时
前端显示材料卡片 "出题中..."
  ↓
后端 generate 接口：
  1. 调用 LLM 生成 min(chunks*3, 30) 道题
  2. 保存到 Question 表（带 chunk_id）
  3. 返回 {question_count, status: "ok"}
  ↓
前端轮询 GET /api/materials/{id}/status（每 2 秒）
  ↓
收到 status="ready" → 卡片更新为 "N 题可用"
收到 status="error" → 卡片更新为 "出题失败，点击重试"
```

### 出题数量公式

```
count = min(total_chunks * 3, 30)
if count < 1: count = 1
```

| total_chunks | count |
|-------------|-------|
| 1 | 3 |
| 2 | 6 |
| 3 | 9 |
| 5 | 15 |
| 10+ | 30 |

## F2：题目库模式

### 按钮状态矩阵

| 材料状态 | 按钮文案 | 点击行为 | 是否 disabled |
|---------|---------|---------|--------------|
| 0 题，不在出题 | "开始答题" | 提示"暂无题目" | 是 |
| 0 题，出题中 | "出题中..." | 无 | 是 |
| 0 题，出题失败 | "重新出题" | 调用 generate | 否 |
| N 题可用 | "开始答题" | 从 DB 取题开始 | 否 |

### 答题启动流程

```
用户点击"开始答题"
  ↓
前端调用 GET /api/materials/{id}/questions?count=N&tags=tag1,tag2&exclude_answered=true
  ↓
后端：从 Question 表中随机取 N 道题
  ↓
返回题目列表（含 id, question, options, answer, explanation, tags）
  ↓
前端直接进入答题阶段（0 秒等待）
```

### 数量选项生成

```
输入：total_questions（题目库总数）, selected_tags（选中标签）
输出：count_options（可选数量按钮）

if selected_tags 不为空:
    available = 该标签的题目数
else:
    available = total_questions

步长规则：
  available ≤ 5  → step = 1
  available ≤ 10 → step = 2
  available > 10 → step = 5

count_options = []
for n from step to available, step step:
    count_options.push(n)
if count_options 为空 or count_options[0] > 1:
    count_options.unshift(1)

默认 = min(5, available)
```

## F3：题目补充

### 逻辑流程

```
用户点击"补充题目"
  ↓
检查 covered_chunks < total_chunks?
  ├─ 是 → 调用 POST /api/materials/{id}/generate?exclude_covered=true
  │       ↓
  │     后端从新 chunk 出题
  │       ↓
  │     更新题目库
  └─ 否 → 显示"已无新题可出"，按钮禁用
```

### 状态

| 条件 | 按钮 | 点击 |
|------|------|------|
| covered < total | "补充题目" | 调用 generate，仅新 chunk |
| covered = total | "已无新题可出"（disabled） | 无 |
| generating | "补充中..."（disabled） | 无 |

## F4：正确率双维度

### 计算规则

```
累计正确率 = 所有 Attempt 中 is_correct 的比例
近期正确率 = {
    if total_attempts < 10:
        最近 3 个 session 的 Attempt 中 is_correct 的比例
    else:
        最近 30 天的 Attempt 中 is_correct 的比例
}
```

### 箭头规则

| 近期 vs 累计 | 显示 |
|-------------|------|
| 近期 > 累计 + 5% | 绿色 ↑ |
| 近期 < 累计 - 5% | 灰色 ↓ |
| 差值 ≤ 5% | 无箭头 |

### 展示位置

```
材料标题行右侧：
┌──────────────────────────────┐
│  材料标题         总 45%    │
│                   近期 80% ↑ │
└──────────────────────────────┘
```

## F5：错题重做闭环

### 状态机

```
        ┌─────────────────────────┐
        │      待攻克 (streak=0)    │ ← 答错进入
        │      错题重做中           │
        └───────┬─────────────────┘
                │ 做对
                ↓
        ┌─────────────────────────┐
        │      待攻克 (streak=1)    │
        │      错题重做中           │
        └───────┬─────────────────┘
                │ 做对
                ↓
        ┌─────────────────────────┐
        │      待攻克 (streak=2)    │
        │      错题重做中           │
        └───────┬─────────────────┘
                │ 做对
                ↓
        ┌─────────────────────────┐
        │      已攻克 ✓            │
        │      移出待攻克列表       │
        │      可在已攻克页查看      │
        └─────────────────────────┘

任何一步做错 → streak 重置为 0 → 回到"待攻克 (streak=0)"
```

### 数据模型

Question 表新增字段：
- `challenge_streak` int default 0
  - 做对 +1
  - 做错重置为 0
  - 达到 3 → 题目标记为"已攻克"
- `mastered` bool default false
  - challenge_streak >= 3 时设为 true

### 错题列表查询

```
GET /api/materials/{id}/wrong-questions
返回：所有 question.attempts 中有 is_correct=0 且 mastered=false 的题目
按 challenge_streak ASC 排序（streak 低的优先做）
```

## F6：间隔复习闭环

### 状态机

```
        ┌─────────────────────────┐
        │      活跃 (is_stable=0)   │
        │      SM-2 管理排期        │
        └───────┬─────────────────┘
                │ interval > 21 天 AND repetitions >= 3
                ↓
        ┌─────────────────────────┐
        │      稳定 (is_stable=1)   │
        │      不再到期提醒         │
        │      可手动复习           │
        └───────┬─────────────────┘
                │ 手动复习
                ↓
        ┌─────────────────────────┐
        │      活跃 (is_stable=0)   │
        │      重新进入 SM-2 循环   │
        └─────────────────────────┘
```

### 到期判定

```
到期 = next_review <= now AND is_stable = false AND total_reviews >= 1
```

（total_reviews >= 1 排除刚创建从未复习的卡片）

### 毕业判定

```
每次 recordReview 后检查：
if interval_days > 21 and repetitions >= 3:
    is_stable = True
```

### ReviewCard 新增字段

- `is_stable` bool default false

## F9：标签选题

### 交互

```
标签云（多选）：
[不限] [AI能力边界] [RAG架构] [Prompt工程化] [评估体系] [数据飞轮]
  ↑ active     ↑ selected    ↑ hover

点击标签 → 切换选中状态
不限 → 清除所有选中

选中标签后：
- 全部题目区实时过滤
- 数量选项自动调整上限
- 开始答题只抽取该标签题目
```

### 后端查询

```
GET /api/materials/{id}/questions?tags=tag1,tag2
→ 返回包含 tag1 AND tag2 的题目
```
