# v4.0 验收清单

> 每个验收项必须实测通过。测试环境：本地 localhost:3000 + localhost:8000，Chrome 无痕窗口。

## F1：上传自动出题

### 正常流程

- [ ] **F1-01** 上传含 200+ 字文本的材料 → 材料卡片立即出现，题目区显示"出题中..."
- [ ] **F1-02** 出题完成后，材料卡片自动更新为"N 题可用"
- [ ] **F1-03** 出题过程中用户可正常浏览其他材料、查看已生成的题目
- [ ] **F1-04** 出题过程中不关闭页面的情况下切换材料再切回来，状态保留
- [ ] **F1-05** 出题过程中，材料卡片显示"出题中..."（由于 Thread 方案不支持中间进度推送，仅显示文案+动画，不显示具体数字进度）

### 数量规则

- [ ] **F1-06** 1 chunk 材料 → 生成 3 题
- [ ] **F1-07** 3 chunk 材料 → 生成 9 题
- [ ] **F1-08** 10+ chunk 材料 → 生成上限 30 题

### 异常处理

- [ ] **F1-09** 出题 API 超时/失败 → 卡片显示"出题失败，点击重试"
- [ ] **F1-10** 点击"重试" → 重新调用出题 API，进度重置
- [ ] **F1-11** 出题失败不影响已生成的题目（如有部分成功）

### 上传反馈

- [ ] **F1-12** 上传成功后显示"已拆分为 N 个知识点块"
- [ ] **F1-13** chunk_count = 0 时显示"内容较短未拆分知识点块"
- [ ] **F1-14** 上传完成后 1.5 秒自动关闭弹窗

## F2：题目库模式

### 空闲态

- [ ] **F2-01** 材料有题目时，按钮文案为"开始答题"（不是"开始锻造"）
- [ ] **F2-02** 材料无题目且正在出题时，按钮显示"出题中..."并禁用
- [ ] **F2-03** 材料无题目且出题失败时，按钮显示"重新出题"
- [ ] **F2-04** 点击"开始答题"后直接显示题目，无"生成中"动画，无 SSE 流式等待

### 数量选择

- [ ] **F2-05** 数量选项基于题目库实际数量动态生成（步长自适应：≤5 步长1，≤10 步长2，>10 步长5）
- [ ] **F2-06** 题目库 3 题 → 可选 [1, 2, 3]
- [ ] **F2-07** 题目库 15 题 → 可选 [5, 10, 15]
- [ ] **F2-08** 默认数量 = min(5, 题目总数)

### 标签选题

- [ ] **F2-09** 选中标签后，数量上限 = 该标签的题目数
- [ ] **F2-10** 选中标签但该标签只有 2 题 → 数量上限为 2，提示"该知识点仅 2 题"
- [ ] **F2-11** 未选标签 → 从全部题目中随机抽

### 题目抽选

- [ ] **F2-12** 每次答题从题目库随机抽取（不是顺序）
- [ ] **F2-13** 同一轮不走重复题
- [ ] **F2-14** 不同轮可以有重复题（除非题库已无新题）

## F4：正确率双维度

### 展示

- [ ] **F4-01** 材料页始终显示"总 N% · 近期 N%"
- [ ] **F4-02** 近期正确率 > 总正确率 → 近期数字绿色 + 箭头 ↑
- [ ] **F4-03** 近期正确率 < 总正确率 → 近期数字灰色 + 箭头 ↓

## F5：错题重做（独立闭环）

### 入口与列表

- [ ] **F5-01** 材料页显示"错题重做（N题待攻克）"入口
- [ ] **F5-02** N = 当前待攻克的错题数（streak < 3）
- [ ] **F5-03** 点击进入 → 显示所有待攻克题目列表
- [ ] **F5-04** 已攻克的题不在列表中（但可在"已攻克"子页查看）

### 答题

- [ ] **F5-05** 错题重做模式下，不显示正确答案（选前选后都不显示）
- [ ] **F5-06** 选对 → 显示"✓ 正确"（绿色），不标注哪个选项是对的
- [ ] **F5-07** 选错 → 显示"✗ 错误"（红色），不标注哪个选项是对的
- [ ] **F5-08** 已选择后不可更改，按钮置灰
- [ ] **F5-09** 显示"下一题"按钮
- [ ] **F5-10** 有键盘快捷键：1-4 选答案，Enter 下一题

### 攻克逻辑

- [ ] **F5-11** 连续第 1 次做对 → streak = 1
- [ ] **F5-12** 连续第 2 次做对 → streak = 2
- [ ] **F5-13** 连续第 3 次做对 → streak = 3 → 标记"已攻克" → 从待攻克列表移除
- [ ] **F5-14** 已攻克时显示"✓ 已攻克！"动画
- [ ] **F5-15** 任何一次做错 → streak 重置为 0，保留在待攻克列表

### 统计

- [ ] **F5-16** 每轮完成后显示"本轮：正确 M / 共 N"
- [ ] **F5-17** 显示"已攻克 +X 题"（本轮新攻克的）

### 跨 session

- [ ] **F5-18** 关闭页面后 streak 数据保留
- [ ] **F5-19** 同一道题在不同 session 的 streak 连续累计

## F6：间隔复习（SM-2 独立闭环）

### 入口与列表

- [ ] **F6-01** 材料页显示"间隔复习（N题到期）"入口
- [ ] **F6-02** N = next_review ≤ now 且 is_stable = false 的卡片数
- [ ] **F6-03** 列表按到期时间排序，最近到期的在前

### 复习流程

- [ ] **F6-04** 初始不显示答案
- [ ] **F6-05** 点击"查看答案" → 正确答案高亮
- [ ] **F6-06** 评分：忘记了(0) / 有印象(1) / 记住了(2) / 很简单(3)
- [ ] **F6-07** 评分后 SM-2 更新间隔
- [ ] **F6-08** 评分 0 或 1 → 明天再复习；评分 2 或 3 → 间隔拉长

### 毕业逻辑

- [ ] **F6-09** interval > 21 天 且 repetitions >= 3 → is_stable = true → 不再出现在到期列表
- [ ] **F6-10** 仅 interval > 21 天但 repetitions < 3 → 不毕业（连续成功次数不够）
- [ ] **F6-11** is_stable = true 的卡片可在"稳定卡片"中查看
- [ ] **F6-12** 用户在稳定卡片中手动复习 → is_stable 重置为 false，interval=1 → 重新进入循环

### 统计

- [ ] **F6-13** 显示"已复习 N 次 · 连续正确 M 次"
- [ ] **F6-14** total_reviews 永远递增，不会重置
