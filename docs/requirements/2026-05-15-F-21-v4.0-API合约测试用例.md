## 1. POST /api/materials

### TC-API-01：文本上传成功
```
POST /api/materials
Content-Type: multipart/form-data
title=测试材料&content=AI产品经理的核心能力包括理解大模型能力边界、设计RAG架构、工程化Prompt系统。这些能力与传统PM需求分析能力互补。
Expected: 200, {"id": 1, "title": "测试材料", "chunk_count": 1}
```

### TC-API-02：内容过短被拒
```
title=测试&content=AI是缩写。
Expected: 400, {"detail": "内容过短，至少需要 50 字"}
```

### TC-API-03：刚好 50 字通过
```
title=测试&content=<正好50字的内容>
Expected: 200, chunk_count >= 0
```

### TC-API-04：缺标题
```
content=有效内容50字以上...
Expected: 422 (FastAPI 自动验证)
```

### TC-API-05：空内容（无 content/url/file）
```
title=测试
Expected: 400, {"detail": "请提供材料内容、网页链接或文件"}
```

### TC-API-06：chunk_count 在响应中
```
正常上传后
Expected: 响应 JSON 包含 "chunk_count" 字段，值为整数
```

## 测试 fixtures

```python
# tests/api/conftest.py
import pytest
import httpx

API_BASE = "http://localhost:8000/api"

@pytest.fixture
def client():
    return httpx.Client(base_url=API_BASE, timeout=30)

@pytest.fixture
def material_id(client):
    """创建测试材料"""
    resp = client.post("/materials", data={
        "title": "API 测试材料",
        "content": "AI产品经理的核心能力包括理解大模型能力边界、设计RAG架构、工程化Prompt系统、建立评估体系和构建数据飞轮。" * 3
    })
    assert resp.status_code == 200
    return resp.json()["id"]

@pytest.fixture
def questions_ready(client, material_id):
    """等待出题完成并返回题目"""
    client.post(f"/materials/{material_id}/generate")
    # 轮询等待
    import time
    for _ in range(30):
        status = client.get(f"/materials/{material_id}/status").json()
        if status["status"] == "ready":
            break
        time.sleep(2)
    return client.get(f"/materials/{material_id}/questions").json()
```
