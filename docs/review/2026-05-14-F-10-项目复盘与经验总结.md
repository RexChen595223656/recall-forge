# 2026-05-14 知锻(RecallForge)项目复盘

## 项目概览

从生成式学习工具概念出发，构建"知锻(RecallForge)"，完整实现了：
- 材料上传 + RAG知识库
- AI动态出题（SSE流式）
- 即时答题反馈
- SM-2间隔重复复习
- 多格式输入（文本/URL/PDF/Markdown）

## 关键决策记录

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 技术栈 | Next.js 16 / Python FastAPI | 采用 | 前端流式体验好，后端AI生态强 |

## 踩坑记录

### 1. DeepSeek Anthropic兼容层流式格式差异
- **问题：** 代码刚开始只处理了 `delta.text`，但DeepSeek返回的是 `delta.type=thinking_delta` 和 `delta.type=text_delta` 两种
- **表现：** SSE流式返回了大量空字符串
- **修复：** 区分处理三种delta类型（thinking_delta/text_delta/signature_delta），只向前端推送text_delta
- **教训：** Anthropic兼容 ≠ 完全相同，不同provider的流式格式细节有差异

### 2. Python .format() 与JSON大括号冲突
- **问题：** QUIZ_PROMPT中的JSON示例花括号被Python `.format()` 误解析为格式化占位符
- **修复：** JSON部分的大括号双写 `{{` `}}`
- **教训：** 包含JSON的prompt模板用 `.replace()` 或双花括号转义

### 3. Python 3.14兼容性
- **问题：** pydantic-core在Python 3.14上没有预编译wheel
- **修复：** 系统已预装pydantic，跳过版本锁定，使用兼容版本
- **教训：** Python最新版本可能缺少主流包支持，考虑锁定3.11/3.12

### 4. Next.js 16 + Tailwind v4 配置方式变化
- **问题：** Tailwind v4使用CSS-based `@theme` 配置替代 `tailwind.config.ts`
- **表现：** 原计划的 `tailwind.config.ts` 文件不起作用
- **修复：** 在globals.css中使用 `@theme inline` 定义自定义token
- **教训：** 新版本框架的breaking changes需要即时适配

## 架构亮点

- **前端直接SSE消费：** 不通过state管理中间层，直接流式渲染
- **答题判定本地化：** 提交答案后前端先行判定，后端异步记录，体验零延迟
- **SM-2算法纯函数：** 无副作用，便于测试和复用
- **ChromaDB嵌入式模式：** 零运维成本，适合单人使用场景
