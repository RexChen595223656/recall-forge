# v3.0 锻造台重构 需求分析文档

## 1. 需求背景

- **需求来源：** 产品自检 + 用户反馈
- **背景描述：** v2.0 三页面架构（知识库/刷题/复习）各自独立，用户无法形成"我学了什么、掌握了多少"的统一认知。知识库只是列表，刷题和复习之间无数据联动，核心体验被侧边栏压缩。
- **关联项目：** 当前 v2.0 全部功能保留，在此基础上升级

## 2. 版本目标

- **业务目标：** 将三个孤立页面重构为一个统一的"知识锻造台"，用户在一个视图中完成"选材料→主动回忆→看结果→持续巩固"全流程
- **核心指标：**
  - 新用户 30 秒内体验完整闭环（上传→出题→答题→反馈）
  - 知识库列表每条材料展示掌握度，一目了然
  - 刷题和复习统一为"锻造"，不跳页面
- **约束条件：**
  - 保留现有后端 API 和数据结构
  - 保留五维出题配置能力
  - 保留 SM-2 间隔重复算法
  - 桌面端优先，移动端后续适配

## 3. 需求分析（双轨模型）

### 3.1 传统分析轨道

| 用户角色 | 场景 | 现状 | 痛点 | 定位 | 价值 | 功能支撑 | 数据支撑 |
|---------|------|------|------|------|------|---------|---------|
| 学习者 | 打开应用，想知道今天该学什么 | 打开首页看到材料列表和统计数字 | 不知道每个材料的掌握状态，只能凭记忆 | 首页即仪表盘 | 一眼看到全局学习状态 | 材料卡片展示掌握度+待复习数 | 题库+复习队列数据 |
| 学习者 | 从材料列表进入刷题 | 点击材料→跳转新页面→设置→等待 | 步骤多，等待长，页面跳转打断上下文 | 点击材料直达锻造区 | 不跳页面，上下文不丢 | 锻造区嵌入材料列表旁 | — |
| 学习者 | 刷完题想看结果 | 完成页显示统计，但回到首页无法回看 | 做完一轮就消失了，没有历史 | 锻造区展示本轮结果+历史 | 每次锻造都有记录 | 会话历史卡片 | 答题记录 |
| 学习者 | 需要复习 | 进入独立的复习页，和刷题完全隔离 | SM-2 数据不反馈到刷题策略 | 复习和刷题统一为"锻造" | 同一个入口，两种模式 | 锻造区支持"新题"和"巩固"双模式 | SM-2 复习队列 |
| 知识工作者 | 积累了多个材料 | 材料列表是扁平列表 | 无搜索、无排序、无标签 | 材料列表支持搜索和按状态筛选 | 快速定位目标材料 | 搜索栏+掌握度筛选 | 掌握度计算 |

### 3.2 AI 可行性分析轨道

- **AI 解决还是规则解决：** 掌握度计算用规则（正确率+SM-2数据），材料搜索用字符串匹配，AI 出题逻辑不变
- **数据基础：** 现有数据（材料/题目/答题记录/复习卡片）完全足够支撑掌握度计算和活动历史
- **模型方案：** 无需新增 AI 方案。出题沿用现有 RAG+LLM 方案
- **评估标准：** 掌握度计算是否与用户主观感受一致（人工验证）
- **冷启动策略：** 无数据的材料显示"未开始"，不伪造数据
- **迭代路径：** 先做规则引擎版的掌握度 → 后续可引入 AI 做自适应难度

### 5.2 核心交互变化

**变化 1：不再跳页面**

- 点击材料 → 锻造区加载该材料的锻造面板（不跳 URL）
- 锻造完成 → 锻造区显示结果面板 + 历史记录刷新
- 用户始终在同一页面，左侧材料列表始终可见

**变化 2：材料卡片升级**

```
旧:  [图标] 标题  日期  [刷题→] [删除]
新:  [掌握度进度条] 标题  待复习N题  [▶]
```

- 每条材料显示掌握度百分比（正确数/总题数）
- 显示待复习数量（红色角标）
- 点击任意位置展开锻造区

**变化 3：锻造区三态切换**

```
空闲态 → 展示当前材料信息 + 历史记录 + 开始锻造按钮
设置态 → 五维配置面板（可折叠，保留上次设置）
锻造中 → 生成动画 → 答题 → 反馈 → 结果
```

**变化 4：锻造模式统一**

- "开始锻造"：基于材料出 5 道新题（避免已出过的 chunk）→ 即时反馈/测验模式
- "仅复习薄弱点"：调出该材料相关的到期复习卡片 → SM-2 评分
- 两种模式共用同一套答题/反馈 UI

**变化 5：出题设置简化**

- 五维配置折叠在"高级设置"里
- 默认展示两个快捷按钮：[开始锻造(默认配置)] [复习薄弱点]
- 点击"高级设置"展开五维面板修改配置

**变化 6：全局统计内嵌**

- 锻造区底部常驻全局统计（当前被选中材料的数据 + 全局数据）
- 不需要单独跳转查看

### 5.3 掌握度计算规则

```
掌握度 = 0%（无答题记录）
掌握度 = 正确数 / 总题数 × 100%（有答题记录）
  加权因素：
  - 最近连续答对次数 > 3 → +10%
  - SM-2 间隔 > 7天且最近评分 ≥ 2 → +5%
  - 有待复习题 → -10%
  上限 100%，下限 0%
```

### 5.4 路由设计

- `/` — 锻造台（知识库 + 锻造区）
- `/upload` — 上传材料
- 删除 `/quiz/[id]` 和 `/review` 独立路由
- 材料选择通过 URL query：`/?material=2`

### 验收卡 1：锻造台布局

### 验收卡 2：掌握度计算

| 准确性标准 | 无数据→0%；全对→100%；部分对→按公式计算 |
| 通过条件 | 不同数据状态的材料正确显示不同掌握度 + 进度条颜色 |

### 验收卡 3：锻造模式统一

| 项目 | 标准 |
|------|------|
| 功能点 | "开始锻造"出 5 道新题，"仅复习薄弱点"调到期复习卡片 |

| 项目 | 标准 |
|------|------|

## 7. 数据需求

- **采集需求：** 材料维度统计（每题关联 material_id 已有）
- **埋点需求：** 锻造完成事件（出题数、正确数、耗时）
- **计算需求：** 掌握度定时计算（每次答题后更新）

## 9. v3.1 迭代更新（2026-05-15）

### 9.1 欢迎页 CTA 调整

- **现状：** 空态最显眼按钮是"全局复习"
- **问题：** 新用户应先添加内容而非复习
- **改法：** 主 CTA 改"添加材料"；"全局复习"仅在有待复习卡片时作为辅助按钮展示

### 9.2 查看材料原文

- **现状：** 上传后仅显示标题和统计，看不到原文
- **改法：** 材料卡片或锻造区新增"查看原文"入口，展示完整文本内容
- **API：** 新增 `GET /api/materials/{id}/content` 或直接复用现有 materials 数据

### 9.3 答题历史回顾

- **现状：** 答对的题直接消失，只有错题进 SM-2 队列
- **改法：** 锻造区"最近锻造"区域改为可点击，展开查看每轮的所有题目（含正确和错误）
- **产品价值：** 用户能回顾自己答过的所有题，不只是错题

### 9.4 复习模块重构：SM-2 从"守门人"变"建议者"

- **核心问题：** SM-2 当前控制访问权——不到期不让看。用户做错一道题想立刻复习，但"复习薄弱点"按钮 disabled 或显示 0。用户困惑"为什么不让我看错题"
- **决策：** SM-2 的角色从"门卫"改为"排序建议者"

**具体改法：**

| 维度 | 当前 | 改为 |
|------|------|------|
| 按钮名称 | 复习薄弱点 | 错题集 |
| 可见内容 | 仅到期卡片 | 全部错题（所有 ReviewCard） |
| SM-2 作用 | 控制哪些卡片可见 | 控制排序优先级 |
| 到期卡片 | 排最前，标"建议今日复习" | 同 |
| 未到期卡片 | 完全不可见 | 排在后面，标"3天后建议复习" |
| 顶部栏提醒 | 待复习 N | "今日建议复习 N 题" |
| 用户主动权 | 被动等待到期 | 随时可做错题，SM-2 在后台优化间隔 |

**SM-2 的价值保留：** 仍然在计算最佳复习时机、更新间隔数据、优化长期记忆。只是不再阻止用户访问。

**API 改动：** 现有 `GET /api/review/due?show_all=true` 已返回所有卡片含 `is_due` 字段，错题集直接复用，按 `next_review` 排序即可。

### 9.5 出题去重提示

- **现状：** `exclude_ids` 机制在后台运行，用户不可见。chunk 全部用完后会重复出题
- **改法：**
  - 材料统计 API 新增 `covered_chunks` / `total_chunks` 字段
  - 全部覆盖后，"开始锻造"按钮 disabled，提示"该材料已覆盖全部知识点，建议上传新内容"
  - 用户可选择"允许重复出题"覆盖此限制

### 9.6 材料内容质量检查

- **现状：** 仅 50 字下限
- **问题：** 50 字对出题几乎无用；无效材料→低质量题目→用户对产品失去信心
- **改法：**
  - 上传前展示提取的文本摘要，让用户确认
  - < 200 字：黄色警告"内容较短，可能无法生成足够题目"
  - 上传后分块数 < 3：锻造区提示"内容较少，建议补充更多材料"
  - 无意义乱码检测：纯随机字符/纯数字 → 红色警告"无法提取有效内容"

- [ ] 全局复习按钮仅在有到期卡片时出现
